3.1

Autoregressive Principle

Contents

  • 3.1 Autoregressive Principle

1. Sequential Data

We start with sequential data and defining what it is, because it's the kind of data that autoregressive models are built for.

A data point is treated as a sequence:

$$x = (x_1, x_2, \dots, x_T)$$

Examples:

  • Sentence → each word = one $x_t$
  • Multivariate time series → each timestep = one $x_t$
  • Image → flattened pixel scan (left→right, top→bottom)
Note: here $x_1, \dots, x_T$ = tokens within one sequence (not diffusion-style latents).

2. The General Goal

Before defining AR models specifically, it helps to place them within the broader goal every generative model is chasing:

Estimate $p(x)$ and learn to sample from it.

  • $p(x)$ → the true, unknown data distribution (what actually generated the observed data; never directly accessible, only sampled from)
  • $p_\theta(x)$ → the model distribution (our approximation, controlled by learnable parameters $\theta$)

What differs between families is how $p_\theta(x)$ is parameterized and fit — not the goal itself.

3. Training Objective

With the goal set, next comes the criterion used to fit the model — this stays fixed for every AR variant covered later.

Fit $\theta$ by minimizing KL divergence between the true and model distributions:

$$\theta^{*} = \arg\min_{\theta} \; D_{KL}\big(p(x) \parallel p_\theta(x)\big)$$

Equivalent to maximizing expected log-likelihood:

$$\theta^{*} = \arg\max_{\theta} \; \mathbb{E}_{x \sim p(x)}\big[\log p_\theta(x)\big]$$

(Since $D_{KL} = \mathbb{E}[\log p(x)] - \mathbb{E}[\log p_\theta(x)]$, and the first term doesn't depend on $\theta$.)

In plain terms: maximize the probability the model assigns to the data we've actually observed — i.e., push $p_\theta$ toward giving high likelihood to real examples.

Key distinction: autoregressive models compute the exact likelihood — no lower bound required, unlike latent-variable-based generative approaches.

4. Defining Equation of Autoregressive (AR) Models

With the objective in place, here is the actual definition of an AR model:

$$p_\theta(x) = p_\theta(x_1, \dots, x_T) = \prod_{t=1}^{T} p_\theta(x_t \mid x_{t-1} )$$

  • $\mid$ → "given" (conditional probability)
  • $x_{t-1}$ → everything before position $t$
  • Derived from the chain rule of probability — exact identity, not an approximation

"Autoregressive" → $x_t$ modeled as a function of its own past (regressing on self).

The log-likelihood decomposes exactly, term by term:

$$\log p_\theta(x) = \sum_{t=1}^{T} \log p_\theta(x_t \mid x_{t-1} )$$

5. Evolution of AR Model Types

This section walks through how AR models developed over time. The objective from Section 3 never changes across any of these — only how $p_\theta(x_t \mid x_{t-1} )$ is computed changes, each version solving a limitation of the one before it.

5.1 Classical AR / Linear Prediction

$$x_t = \sum_{i=1}^{p} a_i \, x_{t-i} + \epsilon_t, \qquad \epsilon_t \sim \mathcal{N}(0, \sigma^2)$$

  • $a_i$ → learnable coefficients ($p$ total — the entire parameter set)
  • $p$ → order (hyperparameter, chosen not learned)
  • $x_t$ is an affine transform of Gaussian noise → $x_t$ is Gaussian

Limitation: only linear, Gaussian dependence → cannot model complex data like natural language.

5.2 Neural AR Models

$$p_\theta(x_t \mid x_{t-i}) = \text{NeuralNet}_\theta(x_{t-i})$$

  • Network: MLP, CNN, etc.
  • Trained via MLE + backpropagation

Limitation: no explicit recurrent memory; fixed-size input context.

5.3 RNNs (→ GRU, LSTM)

Hidden-state recursion:

$$h_t = f_\theta(h_{t-1}, x_{t-1}), \qquad x_t = g_v(h_t)$$

  • $h_t$ → compressed memory of all history up to $t-1$
  • $f_\theta, g_v$ → learnable neural functions
  • Parameter sharing across time: same $\theta, v$ reused at every step → enables arbitrary sequence length

Limitation: vanishing gradients — two equivalent views:

  1. Gradient view: the same parameters get multiplied repeatedly through backprop-through-time → gradient magnitude shrinks for earlier tokens in long sequences.
  2. Bottleneck view: $h_t$ is a single fixed-size vector forced to compress all history $(x_0, \dots, x_{t-1})$ → information loss grows with sequence length.

→ Motivates attention mechanisms (next topic).

6. Summary Table

To close, here's the whole progression side by side:

Model Form of $p_\theta(x_t \mid x_{t-1} )$ Limitation
Classical AR Linear combination of last $p$ values + Gaussian noise Linear, Gaussian only
Neural AR MLP/CNN output No recurrent memory, fixed context
RNN (GRU/LSTM) Hidden-state recursion, shared params Vanishing gradients, memory bottleneck

Constant throughout: the objective (maximize expected log-likelihood).

Variable throughout: the parameterization of $p_\theta(x_t \mid x_{t-1} )$.
