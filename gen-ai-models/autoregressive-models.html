Chapter 3

Autoregressive Models

Next-token prediction, causal attention, and the GPT architecture family.

The Autoregressive Principle

Autoregressive models generate sequences by factorizing the joint probability as a product of conditionals:

$$p(x_1, x_2, \ldots, x_n) = \prod_{i=1}^n p(x_i \mid x_1, x_2, \ldots, x_{i-1})$$

Each token is predicted based only on previous tokens, never future ones. This causal structure is what enables generation.

Causal Attention

The attention mechanism in autoregressive models must be causal (masked) to prevent information from future positions from influencing past predictions.

The attention weight matrix is upper-triangular, with zeros above the diagonal: $$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}} + M\right)V$$ where $M$ is a mask that sets future positions to $-\infty$.

GPT Architecture

  • Decoder-only transformer — uses masked multi-head attention
  • Large context windows — typically 1K-8K+ tokens for ability to condition on extensive history
  • Pre-norm residual connection — LayerNorm applied before attention/MLP blocks
  • No bidirectional context — strictly left-to-right prediction

Key Variants

  • GPT-1: 117M parameters, demonstrated zero-shot task transfer
  • GPT-2: 1.5B parameters, showed strong few-shot capabilities
  • GPT-3: 175B parameters, introduced in-context learning
  • GPT-4: Multimodal, improved reasoning and alignment

Limitations

  • Slow inference — cannot parallelize token generation
  • Error accumulation — mistakes early in generation compound in later tokens
  • Context window limits — bounded history affects long-term coherence
  • Bias modeling — learns to reproduce training data biases, requiring alignment work
