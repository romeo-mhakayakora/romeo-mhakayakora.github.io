Module 02 // Course Notes (Part 2)

Continuing Problems & Gaussian Policies

Never-ending tasks with average reward formulation vs. real-valued actions utilizing parameteric Gaussian distributions.

Two Kinds of "Infinite" — Don't Confuse Them

Everything covered so far (REINFORCE, REINFORCE with Baseline) assumes two things: the episode eventually ends, and the action space is small enough to enumerate (a handful of discrete choices, softmax over them). Both assumptions break down separately, in two completely different ways:

Continuing Problems Continuous Action/State Spaces
What's infinite Time — the task never ends Space — actions/states are real-valued
Symptom $G_t$ sums to infinity Can't enumerate, sum, or argmax over actions
Fix Average reward $r(\pi)$, differential returns Gaussian policy — output $\mu(s,\theta), \sigma(s,\theta)$, sample the action
Example Server-room thermostat (runs forever) Robot arm torque (needs a precise real number)

A task can be episodic with continuous actions, or continuing with discrete actions — these are orthogonal axes. Mixing them up is one of the most common confusions in this material, so we'll treat them as two fully separate problems.

Part 1 — Policy Gradient for Continuing Problems

When tasks never end, total reward sums to infinity. Average Reward replaces discounted return, enabling Actor-Critic to run forever using a differential signal that measures surplus over the mean.

00 — Why Episodic Methods Break

🌡️ The Server Room Example
A climate-control agent manages a server room's temperature. There's no "end" — it runs 24/7, forever. Try the episodic return $G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots$ and the sum runs forever. Even with $\gamma = 0.99$ it takes thousands of steps to decay meaningfully, and $\hat v(s)$ never stabilizes — it just keeps growing.

We can no longer say "the value of this state is the total reward from here to infinity," because that total might genuinely be infinite. We need a different definition of "good."

01 — The New Performance Measure: Average Reward

Instead of the value of a start state, performance becomes the average rate of reward per time step, in the long run:

$$J(\theta) \doteq r(\pi) = \lim_{h\to\infty}\frac{1}{h}\sum_{t=1}^{h}\mathbb{E}[R_t] = \lim_{t\to\infty}\mathbb{E}[R_t] = \sum_s \mu(s)\sum_a \pi(a|s)\sum_{s',r} p(s',r|s,a)\,r$$

Three equivalent readings of the same quantity:

  • Running average: sum rewards over $h$ steps, divide by $h$ — the arithmetic mean, which converges as $h\to\infty$ instead of blowing up.
  • Long-run limit: after running long enough, the expected reward at each step converges to a stable number — the starting state no longer matters (the ergodicity assumption).
  • Explicit sum (most useful form): expected immediate reward for $(s,a)$, weighted by how likely $\pi$ is to pick $a$, weighted by how much time the agent actually spends in $s$ — i.e. $\mu(s)$.

The salary analogy: think of $J(\theta)$ as the agent's hourly wage. In a one-off project (episodic), total earnings matter. In a permanent job (continuing) that never ends, the only sensible question is "how much do I earn per hour, on average?" — that's $r(\pi)$.

02 — Where the Agent Spends Its Time: the Stationary Distribution $\mu(s)$

Computing $r(\pi)$ requires knowing how much time the agent spends in each state — captured by the steady-state distribution:

$$\mu(s) = \lim_{t\to\infty}\Pr\{S_t = s\}$$

Its defining property is self-consistency — flow probability through the policy and environment once, and you get the same distribution back:

$$\sum_s \mu(s)\sum_a \pi(a|s,\theta)\,p(s'|s,a) = \mu(s'), \quad \forall s'$$

Three factors determine $\mu$: current position $\mu(s)$, the agent's choice $\pi(a|s,\theta)$ (the only one it controls), and the world's response $p(s'|s,a)$ (fixed physics). Because changing $\theta$ changes $\pi$, which changes the flow of probability, which changes $\mu$ itself — $\mu$ depends on $\theta$ in a way that's brutally hard to differentiate directly.

🪣 The Fountain Analogy: states are basins at different heights, policy+transitions are the pipes connecting them, $\mu(s)$ is the water level in each basin. Individual water molecules (trajectories) constantly move between basins, but the level in each basin stays constant at equilibrium. Change the pipes (change $\pi$) and the levels shift to a new equilibrium — but wherever you drop the first molecule (start state), it settles to the same equilibrium (ergodicity).

This is exactly why the Policy Gradient Theorem matters: it gives a formula for $\nabla J(\theta)$ that completely avoids computing $\partial\mu/\partial\theta$.

03 — Redefining Return: The Differential Return

Standard return sums to infinity for a continuing task. Fix: subtract the average reward from every single reward before summing.

$$G_t \doteq (R_{t+1} - r(\pi)) + (R_{t+2} - r(\pi)) + (R_{t+3} - r(\pi)) + \cdots$$

💰 Salary Surplus Example: Agent's average is $r(\pi)=\$10$/step. Next three rewards: +\$12, +\$15, +\$10. Standard return: $12+15+10+\cdots = \infty$ — useless. Differential return: $(+2)+(+5)+(0)+\cdots$ — these surpluses shrink toward zero as the agent returns to its typical performance, so the sum converges.

With differential return, value functions become differential value functions: $v_\pi(s) > 0$ means this state is better-than-average going forward, $< 0$ worse-than-average, $0$ exactly typical.

The single algebraic trick that makes everything else work:

Setting Bellman equation for $q_\pi(s,a)$ What changed
Episodic $\sum_{s',r} p(s',r\vert s,a)[r + \gamma v_\pi(s')]$ raw reward $r$, discount $\gamma$
Continuing $\sum_{s',r} p(s',r\vert s,a)[r - r(\pi) + v_\pi(s')]$ differential reward $r-r(\pi)$, no $\gamma$

Replace $r$ with $r - r(\pi)$ and drop $\gamma$ — that's the entire modification. Every downstream structure (Bellman equations, TD errors, the Policy Gradient Theorem proof) survives the jump from episodic to continuing completely intact.

04 — The New Learning Signal: Differential TD Error

Episodic Actor-Critic used $\delta = R + \gamma\hat v(S') - \hat v(S)$. In the continuing case, $\gamma$ disappears and we subtract the running average reward $\bar R$ instead:

$$\delta_t = R_{t+1} - \bar R + \hat v(S_{t+1},\mathbf w) - \hat v(S_t,\mathbf w)$$
  • $R_{t+1} - \bar R$ — immediate surplus: did this exact transition pay better than usual?
  • $\hat v(S_{t+1})$ — future potential: how good does the new state look, going forward?
  • $-\hat v(S_t)$ — old expectation: what did we think the old state was worth? (Subtracted to measure surprise.)

Together: "was this step a good deal?" — immediate pay plus change in outlook, both measured relative to average expectations.

$\bar R$ is itself learned, updated by the same TD error:

$$\bar R \leftarrow \bar R + \alpha_{\bar R}\,\delta$$

The thermostat analogy: $\bar R$ is a thermostat for the learning signal. As the agent improves and earns more per step, $\bar R$ rises automatically to keep $\delta$ centered near zero — preventing $\delta$ from drifting positive forever and pushing the policy in the same direction indefinitely. Only genuinely surprising improvements produce signal.

$\delta$ Meaning Actor response $\bar R$ update
Positive Better than average / better state ↑ prob. of $A_t$ rises
Negative Worse than average / worse state ↓ prob. of $A_t$ falls
Zero Exactly as expected no update unchanged

05 — Proof: The Policy Gradient Theorem (Continuing Case)

Goal: show $\nabla J(\theta)$ reduces to a form using only $\nabla\pi$ and $q_\pi$ — without ever needing $\nabla\mu$.

Step 1 — expand $\nabla v_\pi(s)$ via the product rule, since $v_\pi(s) = \sum_a \pi(a|s)q_\pi(s,a)$:

$$\nabla v_\pi(s) = \sum_a\Big[\nabla\pi(a|s)\,q_\pi(s,a) + \pi(a|s)\,\nabla q_\pi(s,a)\Big]$$

Step 2 — expand $\nabla q_\pi$ using the differential Bellman equation $q_\pi(s,a) = \sum_{s',r}p(s',r|s,a)[r - r(\theta) + v_\pi(s')]$. Since $p(s',r|s,a)$ doesn't depend on $\theta$, $\nabla$ passes through freely.

Step 3 — differentiate the three inner terms:

  • $\nabla r = 0$ (a raw environment number, no $\theta$-dependence)
  • $\nabla(-r(\theta)) = -\nabla r(\theta)$ — the unknown we're solving for; keep it
  • $\nabla v_\pi(s')$ — depends on $\theta$ through the policy from $s'$ onward; keep it

Since $-\nabla r(\theta)$ doesn't depend on $s'$ or $r$, and $\sum_{s',r}p(s',r|s,a)=1$:

$$\nabla q_\pi(s,a) = -\nabla r(\theta) + \sum_{s'}p(s'|s,a)\,\nabla v_\pi(s')$$

Step 4 — substitute back into Step 1 and rearrange to isolate $\nabla r(\theta)$ (using $\sum_a\pi(a|s)=1$ to pull the constant term out cleanly):

$$\nabla r(\theta) = \sum_a\Big[\nabla\pi(a|s)q_\pi(s,a) + \pi(a|s)\sum_{s'}p(s'|s,a)\nabla v_\pi(s')\Big] - \nabla v_\pi(s)$$

Step 5 — the key observation: the left side $\nabla r(\theta) = \nabla J(\theta)$ is a global quantity, independent of $s$. So the right side must evaluate to the same number for any $s$ — which gives us license to sum both sides over all states weighted by $\mu(s)$ (since $\sum_s\mu(s)=1$, the left side is unaffected):

$$\nabla J(\theta) = \sum_s \mu(s)\left[\sum_a\Big[\nabla\pi(a|s)q_\pi(s,a) + \pi(a|s)\sum_{s'}p(s'|s,a)\nabla v_\pi(s')\Big] - \nabla v_\pi(s)\right]$$

Step 6 — split into three pieces:

  • Piece A (what we want): $\sum_s\mu(s)\sum_a\nabla\pi(a|s)q_\pi(s,a)$
  • Piece B: $\sum_s\mu(s)\sum_a\pi(a|s)\sum_{s'}p(s'|s,a)\nabla v_\pi(s')$
  • Piece C: $-\sum_s\mu(s)\nabla v_\pi(s)$

In Piece B, reorder the sums to bring $\sum_{s'}$ to front. The inner part $\sum_s\mu(s)\sum_a\pi(a|s)p(s'|s,a)$ is exactly the stationarity equation from §02 — it equals $\mu(s')$. So Piece B collapses to $\sum_{s'}\mu(s')\nabla v_\pi(s')$.

Step 7 — the great cancellation. Piece B and Piece C are the same sum (dummy variables $s$ and $s'$ both range over the full state space) with opposite signs:

$$\text{Piece B} + \text{Piece C} = \sum_{s'}\mu(s')\nabla v_\pi(s') - \sum_s\mu(s)\nabla v_\pi(s) = 0$$

The impossible-to-compute $\nabla v_\pi$ terms destroy each other completely, leaving:

$$\boxed{\nabla J(\theta) = \sum_s \mu(s)\sum_a \nabla\pi(a|s,\theta)\,q_\pi(s,a)}$$

Identical in form to the episodic Policy Gradient Theorem — just with $\mu(s)$ as the stationary distribution instead of on-policy visitation. It requires only quantities the agent can estimate by sampling: states it visits, its own policy's gradient, and action values.

Converting to a usable update. Turn the sum into an expectation (states arrive naturally at frequency $\mu(s)$), then apply the log-derivative trick ($\nabla\pi = \pi\nabla\ln\pi$) exactly as in vanilla REINFORCE:

$$\nabla J(\theta) = \mathbb{E}_\pi\Big[\nabla\ln\pi(A|S,\theta)\,q_\pi(S,A)\Big] \;\;\Rightarrow\;\; \theta \leftarrow \theta + \alpha^\theta\,\delta\,\nabla\ln\pi(A|S,\theta)$$

where $\delta$ (the differential TD error from §04) stands in for the unknown $q_\pi(S,A)$.

06 — Algorithm 1: One-Step Actor-Critic (Continuing)

Input: differentiable π(a|s,θ), differentiable v̂(s,w)
Parameters: step sizes α^θ, α^w, α^R̄ > 0
Initialize: θ, w, R̄ ← 0.  S ← s0

Loop forever (each time step):
    A ~ π(·|S,θ)
    Take A, observe S', R
    δ ← R − R̄ + v̂(S′,w) − v̂(S,w)        # differential TD error, no γ
    R̄ ← R̄ + α^R̄ · δ                     # update average reward estimate
    w ← w + α^w · δ · ∇v̂(S,w)             # Critic update
    θ ← θ + α^θ · δ · ∇ln π(A|S,θ)        # Actor update
    S ← S′

Differences from episodic one-step AC:
- Gone: $\gamma$ and the discount accumulator $I=\gamma^t$ — no episode means no notion that step 0 matters more than step 100.
- Gone: the "while S is not terminal" inner loop — a single flat loop, forever.
- New: $\bar R$, a third learnable quantity tracking average performance. Updated before Critic and Actor, since it shapes the $\delta$ they both use.

07 — Algorithm 2: Actor-Critic with Eligibility Traces (Continuing)

Input: differentiable π(a|s,θ), differentiable v̂(s,w)
Parameters: λ^w, λ^θ ∈ [0,1], α^w, α^θ, α^R̄ > 0
Initialize: R̄, w, θ ← 0.  S ← s0.  z^w ← 0.  z^θ ← 0

Loop forever (each time step):
    A ~ π(·|S,θ)
    Take A, observe S', R
    δ ← R − R̄ + v̂(S′,w) − v̂(S,w)
    R̄ ← R̄ + α^R̄ · δ
    z^w ← λ^w · z^w + ∇v̂(S,w)             # fade + stamp Critic trace
    z^θ ← λ^θ · z^θ + ∇ln π(A|S,θ)        # fade + stamp Actor trace
    w ← w + α^w · δ · z^w
    θ ← θ + α^θ · δ · z^θ
    S ← S′

Only changes vs. Algorithm 1: initialize $z^w=z^\theta=0$ once, before the loop (never reset — no episode boundaries exist); update both traces each step; replace $\nabla\hat v$ and $\nabla\ln\pi$ in the weight updates with the traces. Also note: unlike the episodic traces version, there's no $I=\gamma^t$ factor in the Actor trace, since nothing is discounted.

One-sentence difference between the two algorithms: one-step credits only the current gradient (one state, one action, one update); traces credit all recently active gradients — a whole recent history, each fading by $\lambda$ per step. The computation per step is almost identical; only what gets multiplied by $\delta$ changes.

  • Critic trace — fade old trace by $\lambda^w$, stamp in the fresh gradient. A state visited 5 steps ago contributes $(\lambda^w)^5$ of its original weight.
  • Actor trace — same idea for $\nabla\ln\pi$. A single positive $\delta$ boosts the probability of the current action and recent actions, with decaying weight — multi-step credit assignment, instead of waiting for the value function to slowly back-propagate credit.

08 — Episodic vs. Continuing: Full Comparison

Episodic Continuing
Objective $J(\theta)$ Value of start state $v_\pi(s_0)$ Average reward rate $r(\pi)$
Return $G_t$ Discounted: $\sum_k\gamma^k R_{t+k+1}$ Differential: $\sum_k(R_{t+k+1}-r(\pi))$
TD error $\delta$ $R+\gamma\hat v(S')-\hat v(S)$ $R-\bar R+\hat v(S')-\hat v(S)$
Discount $\gamma$ Present everywhere Absent entirely
Extra learnable quantity None $\bar R$
$I=\gamma^t$ factor in Actor Yes No
Trace initialization Reset at start of every episode Initialize once, never reset
Loop structure Outer: episodes. Inner: steps to terminal Single flat loop, forever

Keep these separate forever: the continuing/episodic distinction is about time horizon — does the task end? If not: use average reward, drop $\gamma$, add $\bar R$, never reset traces. This is a completely different axis from the continuous/discrete distinction, which is about representation — can you build a lookup table? That's the subject of Part 2.

Part 2 — Policy Parameterization for Continuous Actions

When the action space is infinite, you cannot enumerate probabilities. Instead, the network learns the parameters of a probability distribution — mean and standard deviation — and the agent samples its action from that distribution at every step.

⚠️ This is about action space, not time horizon. Part 1 used Average Reward to handle tasks that never terminate (infinite time). This part handles tasks where actions are real-valued (infinite space). A task can be episodic with continuous actions, or continuing with discrete actions — these axes are fully independent.

00 — Why Every Previous Method Breaks

Failure 1 — The Argmax Problem (Q-learning, DQN). Value-based action selection is $a^* = \arg\max_a Q(s,a)$. With actions $\{$Left, Right, Jump$\}$, that's three computations, done. With a continuous steering angle in $[-180°,+180°]$, there are infinitely many candidate values — argmax over an infinite set never terminates. Discretizing (e.g. every 1°) trades away precision: the true optimum might be 14.573°, and rounding to 14° or 15° produces jerky motor control or lost money in high-frequency trading.

Failure 2 — The Infinite Summation Problem (softmax REINFORCE). Discrete softmax is $\pi(a|s,\theta) = e^{h(s,a,\theta)}/\sum_b e^{h(s,b,\theta)}$. With 4 discrete actions the denominator has 4 terms. With continuous actions it becomes $\int_{-\infty}^\infty e^{h(s,a,\theta)}\,da$ — generally no closed form, uncomputable on every forward pass. Worse, softmax treats each action as an unrelated category (like "Dog" vs "Cat") — it has no notion that 10.1° and 10.2° are related, so it would need infinite neurons and infinite data.

Failure 3 — Curse of Dimensionality (tabular RL). A table needs one entry per state. Ten continuous state variables discretized to 100 bins each gives $100^{10}=10^{20}$ entries — more than the atoms in a human body.

Root cause, all three: these methods are built around enumeration — listing, summing, or maximizing over every possible action or state. Continuous spaces make enumeration impossible. The fix isn't to patch enumeration — it's to abandon it and go generative: instead of picking from a list, generate an action by sampling from a learned distribution.

01 — The Key Idea: Learn Statistics, Not Probabilities

Old (softmax) New (Gaussian)
Network outputs One probability per action: $P(\text{Left}), P(\text{Right}), \ldots$ Two numbers: $\mu$ (best guess), $\sigma$ (uncertainty)
Scales with Number of discrete actions $N$ Number of action dimensions (not action count)
Breaks when $N=\infty$ Never — a 6-DoF robot arm needs 12 outputs total, not millions

Multiple-choice vs. fill-in-the-blank: Softmax is a 4-button exam — assign a probability to each button. Gaussian is fill-in-the-blank: "write the exact steering angle." Instead of a probability per possible number, the policy says "my best guess is 12.5°, and I'm 2° uncertain" — the agent then samples from that bell curve. Two numbers, regardless of required precision.

One distribution per state, not one global distribution. The notation $\pi(a|s,\theta)$ is conditioned on $s$ — every state gets its own bell curve, generated by the same weights processing different inputs.

🚗 Self-driving car example: Straight road → $\mu=0°,\sigma=1°$ (confident, go straight). Sharp curve → $\mu=+30°,\sigma=3°$ (shifted, slightly less certain). Icy road → $\mu=5°,\sigma=8°$ (wide spread — high uncertainty, more exploratory sampling).

02 — The Gaussian PDF

$$p(x) \doteq \frac{1}{\sigma\sqrt{2\pi}}\exp\!\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)$$

*(the $\pi$ here is the constant 3.14159..., not the policy — the textbook explicitly flags this collision of symbols)*

  • $\frac{1}{\sigma\sqrt{2\pi}}$ — normalization constant, ensures total area = 1. Wider bell (large $\sigma$) → shorter peak; narrower bell → taller peak.
  • $\exp(\cdot)$ — the bell shape, always positive, maximal (=1) at $x=\mu$, decaying as you move away.
  • $\frac{(x-\mu)^2}{2\sigma^2}$ — distance term: how far $x$ is from $\mu$, in variance-normalized units, squared so both directions are penalized equally.

⚠️ Density ≠ probability: $p(x)$ is density at a point, not the probability of exactly $x$ — in a continuous space, any single exact value has probability zero. Density can exceed 1: with $\sigma=0.1$, the peak is $\approx 3.99$ — valid, it just means probability mass is tightly concentrated. Actual probability requires an interval: $P(a\le x\le b) = \int_a^b p(x)\,dx$.

Parameter change Effect on curve Meaning for the agent
↑ $\mu$ slides right larger action looks better
↓ $\mu$ slides left smaller action looks better
large $\sigma$ wide, flat high uncertainty → more exploration
small $\sigma$ narrow, tall high confidence → less exploration

03 — The Gaussian Policy

$\mu$ and $\sigma$ are no longer fixed constants — they're functions of state and network weights, which is what makes the whole thing learnable:

$$\pi(a|s,\theta) \doteq \frac{1}{\sigma(s,\theta)\sqrt{2\pi}}\exp\!\left(-\frac{(a-\mu(s,\theta))^2}{2\sigma(s,\theta)^2}\right)$$

where $\mu: \mathcal S\times\mathbb R^{d'}\to\mathbb R$ (any real number — no restriction) and $\sigma:\mathcal S\times\mathbb R^{d'}\to\mathbb R^+$ (must stay strictly positive — an architectural constraint we handle next). This whole expression is differentiable w.r.t. $\theta$, which is the entire foundation that lets policy gradient methods backpropagate through it.

04 — Two-Stage Pipeline: State → Action

Stage ① — the neural network (deterministic). State → hidden features → two output heads:

  • $\mu$-head: $\theta_\mu^\top x_\mu(s)$ → any real number
  • $\sigma$-head: $\exp(\theta_\sigma^\top x_\sigma(s))$ → a strictly positive number

Same state + same weights → same $\mu,\sigma$, every time.

Stage ② — stochastic sampling. Sample $a \sim \mathcal N(\mu,\sigma^2)$. In practice via the reparameterization trick:

$$\varepsilon \sim \mathcal N(0,1), \qquad a = \mu + \sigma\cdot\varepsilon$$

Why reparameterize? You can't differentiate through a raw "sample" operation — randomness itself isn't differentiable. Splitting the randomness ($\varepsilon$, treated as a fixed constant during backprop) from the learnable parameters ($\mu,\sigma$) lets gradients flow through $\mu$ and $\sigma$ even though the realized action was random.

$\sigma$ level Behavior Typically occurs
High (e.g. 5.0) wide bell, frequent far-from-mean samples early training — agent knows nothing, discovers reward regions
Medium (e.g. 1.0) moderate spread mid-training — refining while still exploring nearby
Low (e.g. 0.05) narrow, almost always samples $\mu$ late training — converged, precise

05 — Concrete Implementation

$$\theta = [\theta_\mu,\theta_\sigma]^\top,\qquad \mu(s,\theta)\doteq\theta_\mu^\top x_\mu(s),\qquad \sigma(s,\theta)\doteq\exp\big(\theta_\sigma^\top x_\sigma(s)\big)$$

Why $\mu$ is linear: it needs to cover all of $(-\infty,+\infty)$ with no constraint — a linear function does exactly that, and its gradient is trivially the feature vector $x_\mu(s)$.

Why $\sigma$ needs the exponential: $\sigma$ must always be strictly positive. A raw linear output could get pushed negative by gradient updates, which is meaningless for a standard deviation. $\exp(\text{anything}) > 0$ always — the network learns $\log\sigma$ (any real number) internally, and $\exp$ maps it into the valid positive domain at the last step. 

mu = W_mu @ features(s)              # linear, no activation
log_sigma = W_sigma @ features(s)    # linear, no activation
sigma = torch.exp(log_sigma)         # force positive

The payoff: "with these definitions, all the algorithms in this chapter can be applied to learn real-valued actions." Every previous algorithm — REINFORCE, one-step Actor-Critic, Actor-Critic with traces, episodic or continuing — plugs in unchanged. Only the error signal multiplying the gradient changes; the policy's shape doesn't.

06 — The Eligibility Vector

Every policy gradient update has the same structure:

$$\Delta\theta = \underbrace{\alpha}_{\text{learning rate}} \times \underbrace{E}_{\text{error signal (algorithm-specific)}} \times \underbrace{\nabla\ln\pi(a|s,\theta)}_{\text{eligibility vector (Gaussian-specific)}}$$

The eligibility vector splits across $\theta_\mu$ and $\theta_\sigma$:

$$\nabla_{\theta_\mu}\ln\pi(a|s,\theta_\mu) = \frac{a-\mu(s,\theta)}{\sigma(s,\theta)^2}\,x_\mu(s)$$

Plain English: "move the mean toward the action just taken, in proportion to how far it was, normalized by how uncertain we are." If uncertainty ($\sigma$) is high, deviations are expected and weighted less; if low, deviations are surprising and weighted more.

$$\nabla_{\theta_\sigma}\ln\pi(a|s,\theta_\sigma) = \left[\frac{(a-\mu(s,\theta))^2}{\sigma(s,\theta)^2} - 1\right]x_\sigma(s)$$

The $-1$ centers the signal: if the action landed exactly one standard deviation away (typical), the gradient is zero. Only atypical actions move $\sigma$: a paid-off outlier ($\gg 1$ std devs) widens $\sigma$ (explore more around it); a paid-off central action ($\ll 1$ std devs) narrows $\sigma$ (commit, become more precise).

07 — Deriving $\nabla_{\theta_\mu}\ln\pi$ (Exercise 13.4, Part 1)

We differentiate the log of $\pi$, not $\pi$ itself, because the policy gradient theorem needs $\nabla\pi/\pi = \nabla\ln\pi$ (log-derivative identity), and the log collapses the messy normalization constant and exponential into simple additive terms.

Step 1 — take the log:

$$\ln\pi(a|s,\theta) = -\ln\sigma - \ln\sqrt{2\pi} - \frac{(a-\mu)^2}{2\sigma^2}$$

Step 2 — identify which terms contain $\theta_\mu$. $-\ln\sigma$ and $-\ln\sqrt{2\pi}$ don't depend on $\mu$ → vanish. Only $-\frac{(a-\mu)^2}{2\sigma^2}$ survives.

Step 3 — chain rule through $\mu(s,\theta)$. Let $u=a-\mu$, so $\partial u/\partial\mu = -1$:

$$\frac{\partial}{\partial\mu}\left[-\frac{u^2}{2\sigma^2}\right] = -\frac{2u}{2\sigma^2}\cdot(-1) = \frac{a-\mu}{\sigma^2}$$

and since $\mu(s,\theta) = \theta_\mu^\top x_\mu(s)$ is linear, $\nabla_{\theta_\mu}\mu = x_\mu(s)$. Multiplying the two pieces of the chain rule together:

$$\boxed{\nabla_{\theta_\mu}\ln\pi(a|s,\theta_\mu) = \frac{a-\mu(s,\theta)}{\sigma(s,\theta)^2}\,x_\mu(s)}$$

Reading the sign: if $a>\mu$, gradient is positive — multiplied by a positive error signal, $\theta_\mu$ increases, mean shifts toward the action taken. If $a<\mu$, the opposite. If the error signal is negative (bad outcome), all signs flip and the mean shifts away from that action.

08 — Deriving $\nabla_{\theta_\sigma}\ln\pi$ (Exercise 13.4, Part 2)

Same starting log-expression, now differentiating w.r.t. $\theta_\sigma$ (the weights producing $\log\sigma$, since $\sigma=\exp(\theta_\sigma^\top x_\sigma(s))$).

Term 1: $-\ln\sigma$. Since $\sigma=e^z$ with $z=\theta_\sigma^\top x_\sigma(s)$, we get $\ln\sigma = z$ directly — a huge simplification from the exponential parameterization:

$$\nabla_{\theta_\sigma}(-\ln\sigma) = -x_\sigma(s)$$

Term 2: $-\frac{(a-\mu)^2}{2\sigma^2}$. Chain rule through $\sigma$ first:

$$\frac{\partial}{\partial\sigma}\left[-\frac{(a-\mu)^2}{2}\sigma^{-2}\right] = \frac{(a-\mu)^2}{\sigma^3}, \qquad \nabla_{\theta_\sigma}\sigma = \sigma\,x_\sigma(s)$$

Combining: $\frac{(a-\mu)^2}{\sigma^3}\cdot\sigma\,x_\sigma(s) = \frac{(a-\mu)^2}{\sigma^2}\,x_\sigma(s)$

Adding both terms:

$$\boxed{\nabla_{\theta_\sigma}\ln\pi(a|s,\theta_\sigma) = \left[\frac{(a-\mu(s,\theta))^2}{\sigma(s,\theta)^2} - 1\right]x_\sigma(s)}$$

matching §06 exactly.

09 — The Complete Picture

$$\Delta\theta_\mu = \alpha\cdot E\cdot \frac{a-\mu(s,\theta)}{\sigma(s,\theta)^2}\,x_\mu(s), \qquad \Delta\theta_\sigma = \alpha\cdot E\cdot\left[\frac{(a-\mu(s,\theta))^2}{\sigma(s,\theta)^2}-1\right]x_\sigma(s)$$

The eligibility vectors are properties of the Gaussian policy shape — fixed, regardless of algorithm. Only $E$ changes:

Algorithm Error signal $E$ Setting What it measures
REINFORCE $G_t$ Episodic Was the whole episode good overall?
REINFORCE + Baseline $G_t - \hat v(s,w)$ Episodic Better than usual from this state?
One-Step AC (episodic) $R+\gamma\hat v(S')-\hat v(S)$ Episodic Was this one step better than expected?
One-Step AC (continuing) $R-\bar R+\hat v(S')-\hat v(S)$ Continuing Better than average, and a better next state?
AC with traces same $\delta$, via traces Either Same, credited to recent history too

This is the "plug-and-play" property: once you have a Gaussian policy, any policy gradient algorithm can drive it — you just swap in a different $E$.

Scenario $E$ $(a-\mu)$ $\Delta\theta_\mu$ Result
Good reward, action above mean + + increase mean shifts up toward the successful action
Good reward, action below mean + − decrease mean shifts down toward the successful action
Bad reward, action above mean − + decrease mean shifts away from the failed action
Bad reward, action below mean − − increase mean shifts away from the failed action

10 — Softmax vs. Gaussian, Side by Side

Softmax (discrete) Gaussian (continuous)
Network output one preference $h(s,a,\theta)$ per action $\mu(s,\theta)$ and $\sigma(s,\theta)$ — always 2 per action dimension
Eligibility vector $x(s,a) - \sum_b\pi(b\vert s)x(s,b)$ $\frac{a-\mu}{\sigma^2}x_\mu(s)$ and $\left[\frac{(a-\mu)^2}{\sigma^2}-1\right]x_\sigma(s)$
Log-probability $h(s,a,\theta) - \ln\sum_b e^{h(s,b,\theta)}$ — needs a sum over actions $-\ln\sigma-\ln\sqrt{2\pi}-\frac{(a-\mu)^2}{2\sigma^2}$ — no sum needed
Breaks when $N=\infty$ Never — scales to any real-valued precision

11 — Three Independent Axes, Keep Them Separate

  1. Why Gaussian policy? Because the action space is continuous — can't enumerate/sum over infinite actions. Fix: parameterize a distribution, output $\mu,\sigma$, sample.
  2. Why average reward? Because the task is continuing — no terminal state, standard returns diverge. Fix: subtract the average reward so differential returns stay finite.
  3. Which algorithm? REINFORCE / Actor-Critic / PPO / SAC — determines how the error signal $E$ is computed. The Gaussian policy plugs into all of them unchanged.

These are three independent decisions: policy parameterization (Gaussian vs. softmax) is chosen based on the action space; the objective (average reward vs. discounted return) is chosen based on time horizon; the algorithm is chosen based on sample efficiency and stability needs. Any combination of the four — {discrete, continuous} × {episodic, continuing} — is valid, and each needs its own toolset.
