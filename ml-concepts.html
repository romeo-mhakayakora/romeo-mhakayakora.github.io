Preprocessing & Feature Engineering

6 concepts
01 — Preprocessing

Feature Scaling

Normalization (Min-Max Scaling):

x' = (x - x_min) / (x_max - x_min)

Scales features to a fixed range of [0, 1]. Sensitive to outliers since the min/max are extreme values.

Standardization (Z-Score Scaling):

x' = (x - μ) / σ

Centers data around zero with unit variance. Less sensitive to outliers than normalization. Does not bound data to a fixed range.

Why it matters: Many algorithms (SVM, k-NN, neural networks, PCA) assume all features are on a comparable scale. Without scaling, features with large magnitudes dominate distance computations and gradient updates.

Use for: SVM, k-NN, PCA, neural networks, gradient descent Skip for: Tree-based models (random forest, XGBoost)
02 — Preprocessing

Data Splitting (Train / Validation / Test)

Partitioning the dataset into disjoint subsets to prevent information leakage and obtain honest estimates of generalization performance.

  • Training set (~60–70%): Used to fit model parameters.
  • Validation set (~15–20%): Used for hyperparameter tuning and model selection.
  • Test set (~15–20%): Held out until the final evaluation; used only once to estimate real-world performance.

Common splits: 70/15/15, 80/10/10, or 60/20/20 depending on dataset size. For smaller datasets, cross-validation replaces a fixed validation split.

Standard practice: Always split before any preprocessing that uses statistics from the data Never: Use test data for model selection or repeated evaluation
03 — Preprocessing

Encoding Categorical Variables

One-Hot Encoding: Each category becomes a binary column. Suitable for nominal categories with no ordinal relationship.

[cat_A, cat_B, cat_C] → [1,0,0], [0,1,0], [0,0,1]

Label Encoding: Maps each category to an integer (0, 1, 2, …). Implies ordinality; appropriate for tree-based models but problematic for linear models.

Target Encoding: Replaces each category with the mean of the target variable for that category. Powerful but risks target leakage — requires careful cross-validation.

One-Hot: Linear models, neural networks, distance-based models Label Encoding: Tree-based models (RF, XGBoost, LightGBM)
04 — Preprocessing

Handling Missing Data

Common strategies:

  • Deletion: Remove rows (listwise) or columns with missing values. Simple but can discard useful data.
  • Mean / Median / Mode Imputation: Fill missing entries with the column statistic. Fast but reduces variance and ignores correlations.
  • KNN Imputation: Fill using the mean of k-nearest neighbors. Preserves structure but is computationally expensive.
  • Regression Imputation: Predict missing values using a model trained on observed features.
  • Forward / Backward Fill: Used for time-series — propagate the last known value forward.
Simple: Mean imputation for <5% missing Advanced: KNN or MICE for structured missing patterns

Model Selection & Validation

4 concepts
05 — Model Selection

Bias-Variance Tradeoff

The generalization error of a model decomposes into three components:

E[ (ŷ - y)² ] = Bias(ŷ)² + Var(ŷ) + σ² (irreducible error)
  • Bias: Error from overly simplistic assumptions (underfitting). High-bias models miss relevant patterns.
  • Variance: Error from sensitivity to small fluctuations in the training set (overfitting). High-variance models capture noise.
  • Tradeoff: Increasing model complexity reduces bias but increases variance. The goal is to find the complexity level that minimizes total error.
Diagnose: High training error → high bias; large gap between train & test → high variance
06 — Model Selection

Cross-Validation

K-Fold Cross-Validation: Split data into K folds. Train on K−1 folds, evaluate on the held-out fold. Repeat K times. Report average metric.

CV Score = (1/K) Σᵢ₌₁ᴷ Scoreᵢ
  • k=5 or k=10 are standard choices — balance between bias and computational cost.
  • Stratified K-Fold: Preserves class proportions in each fold — essential for imbalanced classification.
  • Leave-One-Out (LOOCV): K = N (one sample per fold). Low bias, high variance, expensive for large N.
K-Fold: Standard model evaluation Stratified: Imbalanced classification LOOCV: Only for very small datasets (N < 100)
07 — Model Selection

Overfitting & Underfitting

Underfitting: Model is too simple to capture the underlying pattern → high bias, high training error, high test error.

Overfitting: Model memorizes training data including noise → low training error, high test error, large generalization gap.

Detection: Monitor the gap between training and validation performance. A widening gap signals overfitting.

Remedies:

  • Overfitting → More data, regularization (L1/L2), reduce model complexity, dropout, early stopping.
  • Underfitting → Increase model complexity, add features, reduce regularization, train longer.
08 — Model Selection

Hyperparameter Tuning

  • Grid Search: Exhaustive search over a predefined set of hyperparameter values. Guaranteed to find the best within the grid, but exponential in number of parameters.
  • Random Search: Sample hyperparameters from a distribution. More efficient than grid search for high-dimensional spaces — finds good configurations faster.
  • Bayesian Optimization: Builds a probabilistic model mapping hyperparameters to performance. Selects the next candidate using an acquisition function (e.g., expected improvement). More sample-efficient but more complex to implement.
TL;DR: Use random search as a baseline; use Bayesian optimization when each evaluation is expensive (deep learning, large datasets).

Regularization & Optimization

3 concepts
09 — Regularization

Regularization (L1, L2, Elastic Net)

Regularization adds a penalty term to the loss function to discourage overly complex models and reduce overfitting.

L1 (Lasso) Regularization:

Loss = Original Loss + λ Σⱼ |wⱼ|

Encourages sparsity — drives some weights to exactly zero, performing automatic feature selection. Effective when many features are irrelevant.

L2 (Ridge) Regularization:

Loss = Original Loss + λ Σⱼ wⱼ²

Shrinks weights proportionally but does not zero them out. Handles multicollinearity well and is differentiable everywhere.

Elastic Net: Combines L1 and L2 penalties. Useful when there are correlated features and you still want sparsity.

Loss = Original Loss + λ₁ Σⱼ |wⱼ| + λ₂ Σⱼ wⱼ²
L1: Feature selection, high-dimensional sparse problems L2: General-purpose, multicollinearity Elastic Net: Correlated features + sparsity
10 — Optimization

Gradient Descent Variants

w ← w − η · ∇L(w)
  • Batch Gradient Descent: Computes gradient over the entire dataset per update. Deterministic but slow for large datasets.
  • Stochastic Gradient Descent (SGD): Computes gradient on a single sample per update. Fast, noisy — helps escape local minima but never fully converges.
  • Mini-Batch Gradient Descent: Compromise — uses a small random batch (e.g., 32–256 samples). Most common in practice. Balances noise and stability.
  • Adam (Adaptive Moment Estimation): Maintains per-parameter learning rates using running averages of gradients and squared gradients. Default choice for deep learning.
Adam: Default for neural networks SGD with momentum: Often generalizes better than Adam for vision tasks
11 — Optimization

Loss Functions

Regression:

  • Mean Squared Error (MSE): L = (1/n) Σ (y − ŷ)² — penalizes large errors quadratically. Sensitive to outliers.
  • Mean Absolute Error (MAE): L = (1/n) Σ |y − ŷ| — robust to outliers, but gradient is constant (less efficient).
  • Huber Loss: Quadratic for small errors, linear for large errors. Combines best of MSE and MAE.

Classification:

  • Cross-Entropy (Log Loss): L = −(y log(ŷ) + (1−y) log(1−ŷ)) — standard for binary and multi-class classification. Penalizes confident wrong predictions heavily.
  • Hinge Loss: L = max(0, 1 − y · ŷ) — used by SVMs. Encourages a margin of separation.

Evaluation & Metrics

3 concepts
12 — Evaluation

Classification Metrics

  • Accuracy: (TP + TN) / (TP + TN + FP + FN) — misleading for imbalanced classes.
  • Precision: TP / (TP + FP) — of predicted positives, how many are actually positive? Important when false positives are costly.
  • Recall (Sensitivity): TP / (TP + FN) — of actual positives, how many were found? Important when false negatives are costly.
  • F1 Score: 2 · (Precision · Recall) / (Precision + Recall) — harmonic mean; balances precision and recall.
  • AUC-ROC: Area under the ROC curve (TPR vs FPR at various thresholds). Threshold-independent. AUC = 0.5 is random, 1.0 is perfect.
TL;DR: For balanced data → Accuracy. For imbalanced data → Precision, Recall, F1. For ranking → AUC-ROC.
13 — Evaluation

Regression Metrics

  • Mean Squared Error (MSE): (1/n) Σ (y − ŷ)² — penalizes large errors. In same units as y².
  • Root Mean Squared Error (RMSE): √MSE — interpretable in original units. Standard metric for regression.
  • Mean Absolute Error (MAE): (1/n) Σ |y − ŷ| — robust to outliers.
  • R² (Coefficient of Determination): 1 − (SS_res / SS_tot) — proportion of variance explained by the model. Can be negative for poorly fitting models.
  • MAPE: (100/n) Σ |(y − ŷ) / y| — relative error in percentage. Undefined when y = 0.
14 — Evaluation

Confusion Matrix

A K×K table comparing actual labels against predicted labels for a classification problem.

Predicted PositivePredicted Negative
Actual PositiveTrue Positive (TP)False Negative (FN)
Actual NegativeFalse Positive (FP)True Negative (TN)

From the confusion matrix, all classification metrics can be derived: accuracy, precision, recall, specificity (TN / (TN + FP)), and F1 score.

Multi-class: Use a one-vs-rest approach for each class, then macro-average or weighted-average the metrics.

Dimensionality Reduction

2 concepts
15 — Dimensionality Reduction

Principal Component Analysis (PCA)

An unsupervised linear technique that projects data onto orthogonal axes (principal components) that capture maximum variance.

X' = X · W   where   W = eigenvectors of Cov(X)

Steps: (1) Center the data, (2) Compute covariance matrix, (3) Eigendecomposition, (4) Sort eigenvalues and select top-k components, (5) Project.

  • Use cases: Visualization (2D/3D), noise reduction, speeding up models, addressing multicollinearity.
  • Limitations: Linear only; principal components are hard to interpret; scaling matters — always standardize before PCA.
Visualization, noise filtering, pre-processing for linear models
16 — Dimensionality Reduction

t-SNE (t-Distributed Stochastic Neighbor Embedding)

A non-linear, probabilistic technique primarily used for visualization of high-dimensional data in 2D or 3D.

  • Models pairwise similarities in high-dimensional space using a Gaussian distribution.
  • Models pairwise similarities in low-dimensional space using a Student-t distribution (heavier tails → avoids crowding).
  • Minimizes the KL divergence between the two distributions via gradient descent.
  • Key hyperparameter: perplexity (typically 5–50) — balances local vs. global structure.
TL;DR: PCA for exploration. t-SNE for visualization. t-SNE is stochastic (run multiple times), non-parametric (can't embed new points), and computationally expensive.
Visualizing clusters, exploring embeddings, qualitative analysis Not suitable for: Feature extraction, inference on new data, preserving global distances

Ensemble Methods

3 concepts
17 — Ensemble

Bagging (Bootstrap Aggregating)

Train multiple models independently on bootstrap samples (random subsets with replacement) of the training data. Aggregate predictions by averaging (regression) or majority voting (classification).

ŷ_ensemble = (1/M) Σᵢ₌₁ᴹ ŷᵢ

Key properties:

  • Reduces variance without increasing bias — ideal for high-variance models (e.g., deep decision trees).
  • Models can be trained in parallel.
  • Random Forest: Bagging applied to decision trees with an additional randomization step — at each split, only a random subset of features is considered.
Random Forest: Strong default for tabular data, handles non-linearity, minimal tuning
18 — Ensemble

Boosting

Train models sequentially, where each new model focuses on correcting the errors of the previous ensemble.

  • AdaBoost: Increases the weight of misclassified samples after each iteration. Combines weak learners (stumps) into a strong classifier.
  • Gradient Boosting (GBM): Each new model fits the residuals (negative gradient) of the current loss function.
  • XGBoost / LightGBM / CatBoost: Optimized implementations with regularization, tree pruning, and GPU support. Dominant in structured-data competitions.
TL;DR: Boosting reduces bias and variance (with proper regularization). More prone to overfitting than bagging — use early stopping and learning rate scheduling.
Gradient Boosting: Tabular data, regression/classification — state of the art Risk: Overfits if too many estimators or too high learning rate
19 — Ensemble

Stacking (Stacked Generalization)

Combine predictions from multiple diverse base models using a meta-learner (a second-level model).

  • Level-0 (Base models): Diverse algorithms (e.g., RF, SVM, k-NN, XGBoost) trained on the original data.
  • Level-1 (Meta model): Trained on the predictions of the base models (optionally + original features).
  • Key detail: Base model predictions must be generated using cross-validation (out-of-fold) to prevent target leakage to the meta model.

Stacking often yields the best performance in competitions but is more complex and less interpretable than simpler ensembles.
