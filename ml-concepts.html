Preprocessing & Feature Engineering

01

Feature Scaling

Normalization (Min-Max Scaling):

$$x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}}$$

Scales features to $[0, 1]$. Sensitive to outliers since min/max are extreme values.

Standardization (Z-Score Scaling):

$$x' = \frac{x - \mu}{\sigma}$$

Centers data around zero with unit variance. Less sensitive to outliers. Does not bound data to a fixed range.

Many algorithms (SVM, k-NN, neural networks, PCA) assume features are on a comparable scale. Without scaling, features with large magnitudes dominate.

Use for: SVM, k-NN, PCA, neural networks, gradient descent Skip for: Tree-based models (RF, XGBoost)
02

Data Splitting (Train / Validation / Test)

Partition data to prevent information leakage and obtain honest generalization estimates.

  • Training (~60–70%): Fit model parameters.
  • Validation (~15–20%): Hyperparameter tuning and model selection.
  • Test (~15–20%): Final evaluation (used only once).

Common splits: 70/15/15, 80/10/10. For smaller datasets, use cross-validation.

Always split before preprocessing that uses data statistics Never use test data for model selection
03

Encoding Categorical Variables

One-Hot Encoding: Each category becomes a binary column.

$$\texttt{[A, B, C]} \to \texttt{[1,0,0], [0,1,0], [0,0,1]}$$

Label Encoding: Maps categories to integers. Implies ordinality—fine for trees, problematic for linear models.

Target Encoding: Replace category with mean target value. Powerful but risks leakage.

One-Hot: Linear models, neural networks, distance-based Label: Tree-based models (RF, XGBoost)
04

Handling Missing Data

  • Deletion: Remove rows/columns with missing values. Simple but wasteful.
  • Mean/Median/Mode Imputation: Fast, reduces variance, ignores correlations.
  • KNN Imputation: Fill using neighbors. Preserves structure, expensive.
  • Forward/Backward Fill: For time-series—propagate last known value.
Mean imputation for <5% missing KNN or MICE for structured patterns

Model Selection & Validation

05

Bias-Variance Tradeoff

Generalization error decomposes as:

$$\mathbb{E}[(\hat{y} - y)^2] = \operatorname{Bias}(\hat{y})^2 + \operatorname{Var}(\hat{y}) + \sigma_{\varepsilon}^2$$
  • Bias: Error from simplistic assumptions (underfitting).
  • Variance: Error from sensitivity to training fluctuations (overfitting).
  • Tradeoff: Higher complexity → lower bias → higher variance. Minimize total error.
High training error → high bias Large train/test gap → high variance
06

Cross-Validation

K-Fold: Split data into $K$ folds. Train on $K-1$, evaluate on held-out fold. Repeat $K$ times.

$$\text{CV}(K) = \frac{1}{K} \sum_{i=1}^K \text{Score}_i$$
  • $k=5$ or $k=10$: Standard choices.
  • Stratified: Preserves class proportions—essential for imbalanced data.
  • LOOCV: $K=N$. Low bias, high variance, expensive.
K-Fold: Standard evaluation Stratified: Imbalanced classification LOOCV: Only for $N < 100$
07

Overfitting & Underfitting

Underfitting: Model too simple → high bias, high training and test error.

Overfitting: Model memorizes noise → low training error, high test error.

Remedies: Overfit → more data, regularization, reduce complexity, early stopping. Underfit → increase complexity, add features, reduce regularization.

08

Hyperparameter Tuning

  • Grid Search: Exhaustive over a grid. Guaranteed optimum within grid, exponential in parameters.
  • Random Search: Sample from distributions. More efficient for high dimensions.
  • Bayesian Optimization: Probabilistic model of performance. Most sample-efficient.
TL;DR: Random search as baseline; Bayesian when evaluations are expensive.

Regularization & Optimization

09

Regularization (L1, L2, Elastic Net)

L1 (Lasso):

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{data}} + \lambda \sum_{j} |w_j|$$

Encourages sparsity—drives weights to zero (feature selection).

L2 (Ridge):

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{data}} + \lambda \sum_{j} w_j^2$$

Shrinks weights, handles multicollinearity, differentiable everywhere.

Elastic Net: Combines L1 + L2.

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{data}} + \lambda_1 \sum_{j} |w_j| + \lambda_2 \sum_{j} w_j^2$$
L1: Feature selection, sparse problems L2: General-purpose, multicollinearity Elastic Net: Correlated features + sparsity
10

Gradient Descent Variants

$$w^{(t+1)} = w^{(t)} - \eta \nabla \mathcal{L}(w^{(t)})$$
  • Batch: Full dataset per update. Deterministic, slow for large data.
  • SGD: Single sample per update. Noisy, escapes local minima, never fully converges.
  • Mini-Batch: 32–256 samples. Most common in practice.
  • Adam: Per-parameter adaptive learning rates. Default for deep learning.
11

Loss Functions

Regression:

  • MSE: $\displaystyle \frac{1}{n}\sum (y_i - \hat{y}_i)^2$ — penalizes large errors quadratically.
  • MAE: $\displaystyle \frac{1}{n}\sum |y_i - \hat{y}_i|$ — robust to outliers.
  • Huber: Quadratic for small errors, linear for large. Best of both.

Classification:

  • Cross-Entropy: $\displaystyle -\frac{1}{n}\sum \big[y_i\log\hat{y}_i + (1-y_i)\log(1-\hat{y}_i)\big]$ — standard for classification.
  • Hinge: $\max(0, 1 - y\hat{y})$ — used by SVMs.

Evaluation & Metrics

12

Classification Metrics

  • Accuracy: $\displaystyle \frac{TP+TN}{TP+TN+FP+FN}$ — misleading for imbalanced data.
  • Precision: $\displaystyle \frac{TP}{TP+FP}$ — how many predicted positives are correct.
  • Recall: $\displaystyle \frac{TP}{TP+FN}$ — how many actual positives were found.
  • F1: $\displaystyle 2\cdot\frac{P\cdot R}{P+R}$ — harmonic mean of precision and recall.
  • AUC-ROC: Area under ROC curve. $0.5$ = random, $1.0$ = perfect.
TL;DR: Balanced → Accuracy. Imbalanced → Precision, Recall, F1. Ranking → AUC-ROC.
13

Regression Metrics

  • MSE: $\displaystyle \frac{1}{n}\sum (y_i - \hat{y}_i)^2$
  • RMSE: $\displaystyle \sqrt{\frac{1}{n}\sum (y_i - \hat{y}_i)^2}$ — interpretable in original units.
  • MAE: $\displaystyle \frac{1}{n}\sum |y_i - \hat{y}_i|$ — robust to outliers.
  • R²: $\displaystyle 1 - \frac{SS_{\text{res}}}{SS_{\text{tot}}}$ — proportion of variance explained.
  • MAPE: $\displaystyle \frac{100}{n}\sum \big|\frac{y_i - \hat{y}_i}{y_i}\big|$ — percentage error.

Dimensionality Reduction

15

Principal Component Analysis (PCA)

Unsupervised linear projection onto orthogonal axes (principal components) that maximize variance.

$$X' = XW \quad \text{where} \quad W = \text{eigenvectors of } \operatorname{Cov}(X)$$

Steps: (1) Center data, (2) Compute covariance, (3) Eigendecomposition, (4) Select top-$k$, (5) Project.

Use cases: Visualization, noise reduction, speeding up models. Limitations: Linear only, components are hard to interpret.

Always standardize before PCA
16

t-SNE

Non-linear, probabilistic technique for visualization of high-dimensional data in 2D/3D.

  • Models similarities in high-d with Gaussians, in low-d with Student-$t$.
  • Minimizes KL divergence via gradient descent.
  • Key hyperparameter: perplexity (5–50).
TL;DR: PCA for exploration. t-SNE for visualization. Stochastic, non-parametric, expensive.
Not for: Feature extraction, inference, global distance preservation

Ensemble Methods

17

Bagging (Bootstrap Aggregating)

Train models independently on bootstrap samples. Aggregate by average (regression) or vote (classification).

$$\hat{y}_{\text{ens}} = \frac{1}{M}\sum_{m=1}^M \hat{y}_m$$
  • Reduces variance without increasing bias.
  • Models train in parallel.
  • Random Forest: Bagging + random feature subsets at each split.
RF: Strong default for tabular data, minimal tuning
18

Boosting

Train models sequentially, each correcting the previous errors.

  • AdaBoost: Increases weight of misclassified samples.
  • Gradient Boosting: Fits residuals (negative gradient).
  • XGBoost / LightGBM / CatBoost: Optimized with regularization, GPU support.
TL;DR: Boosting reduces both bias and variance. More prone to overfitting than bagging.
Gradient Boosting: State of the art for tabular data Risk: Overfits with too many estimators or high learning rate
19

Stacking (Stacked Generalization)

Combine diverse base models via a meta-learner trained on their predictions.

  • Level 0: Diverse algorithms (RF, SVM, k-NN, XGBoost).
  • Level 1: Meta model trained on out-of-fold predictions to prevent leakage.

Often best in competitions, but more complex and less interpretable.
