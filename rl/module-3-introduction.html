Module 03

Advantage Actor-Critic (A2C)

Combining policy gradients with a learned value function: the actor-critic architecture, the advantage function, and how it improves learning.

Contents

  • 03.1 A2C Introduction
  • 03.2 Estimating Advantage
  • 03.3 A2C Algorithm

Overview

  • Actor-Critic algorithms combine policy gradient + a learned value function.
  • A policy is reinforced with a learned reinforcing signal generated using a learned value function.
  • Contrast with REINFORCE: REINFORCE uses a high-variance Monte Carlo estimate of the return to reinforce the policy.
  • Two components, learned jointly:
    • Actor — learns a parameterized policy.
    • Critic — learns a value function to evaluate state-action pairs; provides the reinforcing signal to the actor.

Motivation

  • A learned reinforcing signal can be more informative for a policy than the raw rewards available from an environment.
    • Example: it can transform a sparse reward (e.g. agent only gets +1 upon success) into a dense reinforcing signal.
  • Learned value functions typically have lower variance than Monte Carlo estimates of the return → reduces uncertainty under which a policy learns → makes learning easier.
  • Trade-off: training becomes more complex. Learning the policy now depends on the quality of the value function estimate, which is being learned simultaneously. Until the value function generates reasonable signals, learning good actions is challenging.

The advantage function as the reinforcing signal

It's common to learn the advantage function as the reinforcing signal:

$$A^\pi(s,a) = Q^\pi(s,a) - V^\pi(s)$$

Key idea: it's better to select an action based on how it performs relative to the other actions available in a particular state, instead of using the absolute value of that action (as measured by the Q-function).

The advantage quantifies how much better or worse an action is than the average available action.

Actor-Critic algorithms that learn the advantage function are known as Advantage Actor-Critic (A2C) algorithms.

The Actor

Actors learn parametrized policies $\pi_\theta$ using the policy gradient — very similar to REINFORCE, except the advantage $A_t^\pi$ is used as the reinforcing signal instead of the Monte Carlo return $R_t(\tau)$.

Actor-Critic: $\displaystyle \nabla_\theta J(\pi_\theta) = \mathbb{E}_t\big[A_t^\pi \nabla_\theta \log \pi_\theta(a_t\mid s_t)\big] \tag{3.1}$
REINFORCE: $\displaystyle \nabla_\theta J(\pi_\theta) = \mathbb{E}_t\big[R_t(\tau) \nabla_\theta \log \pi_\theta(a_t\mid s_t)\big] \tag{3.2}$

The only structural difference between the two: $A_t^\pi$ replaces $R_t(\tau)$.

The Critic

Critics are responsible for learning how to evaluate $(s,a)$ pairs, and using this to generate $A^\pi$.

Covered here: (1) the advantage function and why it's a good reinforcing signal, (2) two estimation methods — n-step returns and GAE, (3) how they're learned in practice.

The Advantage Function

Intuitively, $A^\pi(s_t,a_t)$ measures the extent to which an action is better or worse than the policy's average action in a particular state.

$$A^\pi(s_t,a_t) = Q^\pi(s_t,a_t) - V^\pi(s_t) \tag{3.3}$$

Property 1 — zero-mean over actions

$$\mathbb{E}_{a \in \mathcal{A}}[A^\pi(s_t,a)] = 0$$

  • If all actions are essentially equivalent, $A^\pi = 0$ for all of them → probability of taking these actions stays unchanged when the policy is trained using $A^\pi$.
  • Compare to a reinforcing signal based on absolute state/state-action values: this would have a constant value in the same situation, but that constant may not be 0 → it would actively encourage (if positive) or discourage (if negative) the action, even though all actions are equivalent.
  • In practice, this specific case (all actions equivalent) may not be too problematic, though it is unintuitive.

Property 2 — matches intuition when an action is below average but still has positive expected return

  • More problematic example: action taken was worse than the average action, but expected return is still positive — i.e. $Q^\pi(s_t,a_t) > 0$ but $A^\pi(s_t,a_t) < 0$.
  • Ideally, this action should become less likely, since better options were available.
  • Using $A^\pi$ produces behavior matching this intuition (discourages the action).
  • Using $Q^\pi$ alone — or even $Q^\pi$ with a baseline — may incorrectly encourage the action instead.

Property 3 — advantage is a relative measure

  • For a given state $s$ and action $a$, it considers $Q^\pi(s,a)$ and evaluates whether $a$ takes the policy to a better or worse place, relative to $V^\pi(s)$.
  • This avoids penalizing an action for the policy currently being in a particularly bad state, and avoids crediting an action for the policy being in a good state.
  • Rationale: action $a$ can only affect the future trajectory, not how the policy arrived at the current state — so the action should be evaluated based on how it changes value going forward.

Example — same advantage, very different absolute values (good state vs. bad state)

Good state: $Q^\pi(s,a) = 110,\ V^\pi(s) = 100,\ A^\pi(s,a) = 10 \tag{3.4}$
Bad state: $Q^\pi(s,a) = -90,\ V^\pi(s) = -100,\ A^\pi(s,a) = 10 \tag{3.5}$
  • In both cases, action $a$ yields the same relative improvement of 10, captured identically by $A^\pi$ in both cases.
  • This equivalence would not be clear if only $Q^\pi(s,a)$ were considered.

Why the advantage captures long-term effects

The advantage function is able to capture the long-term effects of an action, because it considers all future time steps (within the time horizon implicitly given by $\gamma$) and ignores the effects of all actions taken to date.

Schulman et al. present a similar interpretation in their paper "Generalized Advantage Estimation."

Having established why $A^\pi(s,a)$ is a good reinforcing signal, the next step is to look at two ways of estimating it: n-step returns and Generalized Advantage Estimation (GAE).

Estimating $Q^\pi$ from $V^\pi$

To calculate $A^\pi$, we need estimates of both $Q^\pi$ and $V^\pi$.

Option considered but rejected: learn $Q^\pi$ and $V^\pi$ separately with two different networks. Two disadvantages:

  1. Care is needed to ensure the two estimates stay consistent with each other.
  2. It's less efficient to learn.

Preferred approach: learn just $V^\pi$, and combine it with rewards from a trajectory to estimate $Q^\pi$.

Why $V^\pi$ is preferred over $Q^\pi$:

  1. $Q^\pi$ is a more complex function and may require more samples to learn well — particularly problematic when actor and critic are trained jointly.
  2. Computing $V^\pi$ from $Q^\pi$ is more expensive: it requires computing $Q^\pi(s,a)$ for all possible actions in a state, then taking the action-probability-weighted average. This is also difficult for environments with continuous action spaces.

Rewriting $Q^\pi$ using $V^\pi$ (n-step return)

If we assume a perfect estimate of $V^\pi(s)$ is available, $Q^\pi$ can be rewritten as a mix of expected rewards for $n$ time steps, followed by $V^\pi(s_{t+n+1})$:

$$Q^\pi(s_t,a_t) = \mathbb{E}\big[r_t + \gamma r_{t+1} + \cdots + \gamma^n r_{t+n} + \gamma^{n+1}V^\pi(s_{t+n+1})\big] \tag{3.6}$$

To make this tractable to estimate, substitute a single sampled trajectory of rewards $(r_t, \ldots, r_{t+n})$ in place of the expectation, and substitute in $V^\pi(s_{t+n+1})$ learned by the critic. This is the n-step return:

$$Q^\pi(s_t,a_t) \approx r_t + \gamma r_{t+1} + \cdots + \gamma^n r_{t+n} + \gamma^{n+1}V^\pi(s_{t+n+1}) \tag{3.7}$$

Bias–variance trade-off made explicit by Eq. 3.7:

  • The $n$-step sum of actual rewards is unbiased, but has high variance (comes from just one trajectory).
  • $V^\pi(s_{t+n+1})$ has lower variance (reflects an expectation over all trajectories seen so far), but is biased (estimated via a function approximator).
  • Close to $t$: benefit of using unbiased real rewards typically outweighs the variance they introduce.
  • As $n$ increases: variance from real rewards becomes problematic → switching to the lower-variance, biased value estimate becomes preferable.
  • $n$ = the knob controlling this bias–variance trade-off.

Generalized Advantage Estimation (GAE)

GAE (Schulman et al.) improves on the n-step return estimate for the advantage function. It solves the problem of having to explicitly pick a single value of $n$.

Main idea: instead of picking one $n$, mix multiple values $n = 1, 2, 3, \ldots, k$.

Purpose: significantly reduce estimator variance while keeping the introduced bias as low as possible.

GAE is defined as an exponentially weighted average of all the n-step forward return advantage estimators:

$$A^\pi_{\text{GAE}}(s_t,a_t) = \sum_{\ell=0}^{\infty} (\gamma\lambda)^\ell \delta_{t+\ell} \tag{3.10}$$

$$\text{where} \quad \delta_t = r_t + \gamma V^\pi(s_{t+1}) - V^\pi(s_t) \tag{3.11}$$

Intuitively: GAE takes a weighted average of many different advantage estimates, each with different bias/variance.

  • It starts from the high-bias, low-variance 1-step estimator.
  • It adds contributions from lower-bias, higher-variance estimators using 2, 3, ..., $n$ steps.
  • Each contribution's weight decays exponentially as the number of steps increases; the decay rate is controlled by $\lambda$.
  • Larger $\lambda$ → higher variance (more weight on longer, higher-variance estimators).

Full derivation of Eq. 3.10 (starting from the raw n-step estimators, including the telescoping-cancellation proof) is in a separate file: GAE_Derivation.md.

GAE vs. n-step: why $\lambda$ is a "softer" knob than $n$

  • Both $n$ and $\lambda$ control the same underlying bias–variance trade-off.
  • $n$ is a hard choice — it precisely determines the exact point where high-variance real rewards are swapped out for the low-variance value-function estimate.
  • $\lambda$ is a soft choice — smaller $\lambda$ weights the $V$-function baseline more heavily; larger $\lambda$ weights the actual rewards more heavily. The transition is smooth/blended rather than a hard cutoff.

Learning the Advantage Function (in practice)

Both estimation methods (n-step, GAE) assume access to an estimate $\hat{V}^\pi$:

$$A^\pi_{\text{NSTEP}}(s_t,a_t) \approx r_t + \gamma r_{t+1} + \cdots + \gamma^n r_{t+n} + \gamma^{n+1}\hat{V}^\pi(s_{t+n+1}) - \hat{V}^\pi(s_t) \tag{3.18}$$

$$A^\pi_{\text{GAE}}(s_t,a_t) \approx \sum_{\ell=0}^{\infty}(\gamma\lambda)^\ell \hat{\delta}_{t+\ell}, \quad \text{where} \quad \hat{\delta}_t = r_t + \gamma\hat{V}^\pi(s_{t+1}) - \hat{V}^\pi(s_t) \tag{3.19}$$

(The hats denote estimates, not the true value function.)

The circular dependency

Both advantage estimators depend on having a good $\hat{V}^\pi$. But $\hat{V}^\pi$ is itself learned — and the way it's learned (via TD targets) can itself depend on advantage-related quantities.

This is resolved in practice (e.g. in PPO and similar actor-critic methods) by breaking the circularity within each update:

  1. Run the policy to collect a batch of experience.
  2. Use the current $\hat{V}^\pi$ to compute GAE advantages for that batch.
  3. Use those advantages (and/or returns) to update the policy.
  4. Simultaneously update $\hat{V}^\pi$ — but the value targets used for this update are computed with the fixed (old) value estimates, not the one being updated.

In one sentence: GAE gives a good advantage estimate given a value function; TD learning gives a good value function given experience — the two are trained jointly, with old estimates used to break the circularity within a given update step.

$V^\pi$ is learned with TD learning

Same idea as how DQN learns $Q^\pi$, but applied to $V^\pi$ instead.

Process:

  1. Parametrize $V^\pi$ with neural network weights $\theta$: $\hat{V}^\pi(s;\theta)$.
  2. Generate a target value $V^\pi_{\text{tar}}$ from experience.
  3. Minimize a regression loss (e.g. MSE) between $\hat{V}^\pi(s;\theta)$ and $V^\pi_{\text{tar}}$.
  4. Repeat, updating $\theta$ each time.

Loss:

$$L(\theta) = \mathbb{E}\Big[\big(\hat{V}^\pi(s;\theta) - V^\pi_{\text{tar}}\big)^2\Big]$$

Three ways to construct $V^\pi_{\text{tar}}$:

  1. 1-step TD target:

    $$V^\pi_{\text{tar}} = r_t + \gamma\hat{V}^\pi(s_{t+1};\theta_{\text{old}}) \tag{3.20}$$

  2. Monte Carlo estimate:

    $$V^\pi_{\text{tar}}(s_t) = \sum_{t'=t}^{T} \gamma^{t'-t} r_{t'} \tag{3.21}$$

  3. Using the n-step advantage estimator itself:

    $$V^\pi_{\text{tar}}(s_t) = A^\pi_{\text{NSTEP}}(s_t,a_t) + V^\pi(s_t) \tag{3.22}$$

In practice, the choice of $V^\pi_{\text{tar}}$ is usually tied to the method used to estimate the advantage — e.g. Eq. 3.20 when estimating with n-step returns, or GAE-based targets when using GAE. It's also possible to use more advanced optimization procedures to learn $V^\pi$ — e.g. a trust-region method.

Plain-English intuition: what an n-step advantage really is

  • A 2-step advantage is just the 1-step advantage (TD error) at time $t$, plus a discounted 1-step advantage at time $t+1$:

$$A_t^\pi(2) = \underbrace{\delta_t}_{\text{advantage now}} + \underbrace{\gamma\delta_{t+1}}_{\text{advantage next step, discounted}}$$

  • More generally: an n-step advantage is a sum of 1-step advantages (TD errors), each discounted by $\gamma$ according to how far into the future it is:

$$A_t^\pi(n) = \sum_{i=0}^{n-1}\gamma^i \delta_{t+i}$$

  • Why this works (telescoping): each $\delta_{t+i}$ "hands off" its next-state value estimate to the following $\delta$, and that next $\delta$ "returns" it by using it as its own current-state value. All the intermediate value terms $V^\pi(s_{t+1}), V^\pi(s_{t+2}), \ldots, V^\pi(s_{t+n-1})$ cancel out in pairs (telescope away), leaving only:
    • the initial value estimate $V^\pi(s_t)$ (negative),
    • the final bootstrapped value $\gamma^n V^\pi(s_{t+n})$,
    • and all the rewards in between.
  • Interpretation: each $\delta_{t+i}$ tells you "how much better or worse did things go than expected at that specific step." The n-step advantage simply accumulates these step-by-step surprises, discounting future surprises since they're less certain and farther away.
  • This is why the $\delta$-formulation (GAE) is powerful: it turns a messy expression involving nested value functions into a simple sum of independent TD errors — clean and efficient to implement.
