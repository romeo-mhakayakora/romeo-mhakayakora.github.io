Module 03

Advantage Actor-Critic (A2C)

Combining policy gradients with a learned value function: the actor-critic architecture, the advantage function, and how it improves learning.

Contents

  • 03.1 A2C Introduction
  • 03.2 Estimating Advantage
  • 03.3 A2C Algorithm

Overview

  • Actor-Critic algorithms combine policy gradient + a learned value function.
  • A policy is reinforced with a learned reinforcing signal generated using a learned value function.
  • Contrast with REINFORCE: REINFORCE uses a high-variance Monte Carlo estimate of the return to reinforce the policy.
  • Two components, learned jointly:
    • Actor — learns a parameterized policy.
    • Critic — learns a value function to evaluate state-action pairs; provides the reinforcing signal to the actor.

Motivation

  • A learned reinforcing signal can be more informative for a policy than the raw rewards available from an environment.
    • Example: it can transform a sparse reward (e.g. agent only gets +1 upon success) into a dense reinforcing signal.
  • Learned value functions typically have lower variance than Monte Carlo estimates of the return → reduces uncertainty under which a policy learns → makes learning easier.
  • Trade-off: training becomes more complex. Learning the policy now depends on the quality of the value function estimate, which is being learned simultaneously. Until the value function generates reasonable signals, learning good actions is challenging.

The advantage function as the reinforcing signal

It's common to learn the advantage function as the reinforcing signal:

$$A^\pi(s,a) = Q^\pi(s,a) - V^\pi(s) \tag{3.3}$$

Key idea: it's better to select an action based on how it performs relative to the other actions available in a particular state, instead of using the absolute value of that action (as measured by the Q-function).

The advantage quantifies how much better or worse an action is than the average available action.

Actor-Critic algorithms that learn the advantage function are known as Advantage Actor-Critic (A2C) algorithms.

The Actor

Actors learn parametrized policies $\pi_\theta$ using the policy gradient — very similar to REINFORCE, except the advantage $A_t^\pi$ is used as the reinforcing signal instead of the Monte Carlo return $R_t(\tau)$.

Actor-Critic: $\displaystyle \nabla_\theta J(\pi_\theta) = \mathbb{E}_t\big[A_t^\pi \nabla_\theta \log \pi_\theta(a_t\mid s_t)\big] \tag{3.1}$
REINFORCE: $\displaystyle \nabla_\theta J(\pi_\theta) = \mathbb{E}_t\big[R_t(\tau) \nabla_\theta \log \pi_\theta(a_t\mid s_t)\big] \tag{3.2}$

The only structural difference between the two: $A_t^\pi$ replaces $R_t(\tau)$.

The Critic

Critics are responsible for learning how to evaluate $(s,a)$ pairs, and using this to generate $A^\pi$.

Covered here: (1) the advantage function and why it's a good reinforcing signal, (2) two estimation methods — n-step returns and GAE, (3) how they're learned in practice.

The Advantage Function

Intuitively, $A^\pi(s_t,a_t)$ measures the extent to which an action is better or worse than the policy's average action in a particular state.

$$A^\pi(s_t,a_t) = Q^\pi(s_t,a_t) - V^\pi(s_t) \tag{3.3}$$

Property 1 — zero-mean over actions

$$\mathbb{E}_{a \in \mathcal{A}}[A^\pi(s_t,a)] = 0$$

  • If all actions are essentially equivalent, $A^\pi = 0$ for all of them → probability of taking these actions stays unchanged when the policy is trained using $A^\pi$.
  • Compare to a reinforcing signal based on absolute state/state-action values: this would have a constant value in the same situation, but that constant may not be 0 → it would actively encourage (if positive) or discourage (if negative) the action, even though all actions are equivalent.
  • In practice, this specific case (all actions equivalent) may not be too problematic, though it is unintuitive.

Property 2 — matches intuition when an action is below average but still has positive expected return

  • More problematic example: action taken was worse than the average action, but expected return is still positive — i.e. $Q^\pi(s_t,a_t) > 0$ but $A^\pi(s_t,a_t) < 0$.
  • Ideally, this action should become less likely, since better options were available.
  • Using $A^\pi$ produces behavior matching this intuition (discourages the action).
  • Using $Q^\pi$ alone — or even $Q^\pi$ with a baseline — may incorrectly encourage the action instead.

Property 3 — advantage is a relative measure

  • For a given state $s$ and action $a$, it considers $Q^\pi(s,a)$ and evaluates whether $a$ takes the policy to a better or worse place, relative to $V^\pi(s)$.
  • This avoids penalizing an action for the policy currently being in a particularly bad state, and avoids crediting an action for the policy being in a good state.
  • Rationale: action $a$ can only affect the future trajectory, not how the policy arrived at the current state — so the action should be evaluated based on how it changes value going forward.

Example — same advantage, very different absolute values (good state vs. bad state)

Good state: $Q^\pi(s,a) = 110,\ V^\pi(s) = 100,\ A^\pi(s,a) = 10 \tag{3.4}$
Bad state: $Q^\pi(s,a) = -90,\ V^\pi(s) = -100,\ A^\pi(s,a) = 10 \tag{3.5}$
  • In both cases, action $a$ yields the same relative improvement of 10, captured identically by $A^\pi$ in both cases.
  • This equivalence would not be clear if only $Q^\pi(s,a)$ were considered.

Why the advantage captures long-term effects

The advantage function is able to capture the long-term effects of an action, because it considers all future time steps (within the time horizon implicitly given by $\gamma$) and ignores the effects of all actions taken to date.

Schulman et al. present a similar interpretation in their paper "Generalized Advantage Estimation."

Having established why $A^\pi(s,a)$ is a good reinforcing signal, the next step is to look at two ways of estimating it: n-step returns and Generalized Advantage Estimation (GAE).
