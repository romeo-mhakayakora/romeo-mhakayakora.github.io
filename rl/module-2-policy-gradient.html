Module 02

Policy Gradient

Primer

  • This is a primer on policy gradient methods — the family of algorithms that directly compute the gradient of the objective with respect to policy parameters and use it to improve the policy.
  • Key idea: during learning, actions that led to good outcomes should become more probable (positively reinforced); actions that led to bad outcomes should become less probable.
  • If learning succeeds, over many iterations action probabilities shift toward a distribution that performs well in the environment.
  • The algorithm needs three components, each depending on the one before it:
    • A parametrized policy (a strategy) — maps states to action probabilities.
    • An objective to be maximized — depends on the policy, since it is defined as the expected return generated by that policy.
    • A method for updating the policy parameters — depends on the objective, since it uses the objective's gradient to know how to change the parameters.
  • The policy gradient provides a way to estimate the gradient of the objective with respect to the policy parameters.
  • This is a crucial step, since the policy gradient is used to modify the policy parameters so as to maximize the objective.
  • We are now laying the groundwork for the policy gradient itself — the following sections build up, one at a time, the policy, the objective, and finally the policy gradient.

1.1 Policy

  • A policy $\pi$ is a function mapping states to action probabilities: sample action $a \sim \pi(s)$.
  • Goal: a good policy = one that maximizes cumulative discounted rewards.
  • Method: function approximation via neural networks (powerful, flexible approximators).
  • Policy network: $\pi_\theta$, parametrized by $\theta$ (learnable parameters).
  • Each specific set of parameter values $\theta$ represents a particular policy.
    • If $\theta_1 \neq \theta_2$ then $\pi_{\theta_1}(s) \neq \pi_{\theta_2}(s)$ for some state $s$ → different policies.
    • One neural network architecture can represent many different policies (via different $\theta$).
  • Learning a good policy = searching for a good set of values for $\theta$.
  • Requirement: policy network must be differentiable (mechanism for improvement = gradient ascent in parameter space, covered in 1.3).

1.2 The Objective Function

  • Objective = agent's goal (e.g., win a game, get highest score).
  • Trajectory: $\tau = s_0, a_0, r_0, \ldots, s_T, a_T, r_T$ — sequence of states, actions, rewards generated by agent acting in environment.
  • Return $R_t(\tau)$: discounted sum of rewards from time step $t$ to end of trajectory.

$$R_t(\tau) = \sum_{t'=t}^{T} \gamma^{t'-t} r_{t'} \tag{1.1}$$

  • Sum starts at $t$, but discount factor $\gamma$'s exponent starts at $0$ → offset exponent by $(t'-t)$.
  • At $t=0$: $R_0(\tau) = R(\tau)$ (return of complete trajectory).

$$J(\pi_\theta) = \mathbb{E}_{\tau \sim \pi_\theta}[R(\tau)] = \mathbb{E}_{\tau \sim \pi_\theta}\left[\sum_{t=0}^{T} \gamma^t r_t\right] \tag{1.2}$$

  • Expectation calculated over many trajectories sampled from policy ($\tau \sim \pi_\theta$).
  • Expectation approaches true value as more samples are gathered.
  • $J$ is tied to the specific policy $\pi_\theta$ used.

1.3 The Policy Gradient

  • Two ingredients defined so far: policy $\pi_\theta$, objective $J(\pi_\theta)$.
  • Third/final component: the policy gradient.

Solves:

$$\max_\theta J(\pi_\theta) = \mathbb{E}_{\tau \sim \pi_\theta}[R(\tau)] \tag{1.3}$$

  • To maximize $J$: perform gradient ascent on parameters $\theta$.
    • Gradient points in direction of steepest ascent.
    • Compute gradient, use to update parameters:

$$\theta \leftarrow \theta + \alpha \nabla_\theta J(\pi_\theta) \tag{1.4}$$

  • $\alpha$ = learning rate (scalar); controls size of the parameter update.
  • Parameter update can use any suitable optimizer that takes $\nabla_\theta J(\pi_\theta)$ as input.
  • $J(\pi_\theta)$ can be thought of as an abstract hypersurface on which we search for the maximum point, with $\theta$ as variables. (A hypersurface in $N$ dimensions is an object of $(N-1)$-dimension.)

Policy gradient = the term $\nabla_\theta J(\pi_\theta)$:

$$\nabla_\theta J(\pi_\theta) = \mathbb{E}_{\tau \sim \pi_\theta}\left[\sum_{t=0}^{T} R_t(\tau) \nabla_\theta \log \pi_\theta(a_t \mid s_t)\right] \tag{1.5}$$

  • $\pi_\theta(a_t \mid s_t)$ = probability of the action taken by the agent at time step $t$; action sampled $a_t \sim \pi_\theta(s_t)$.
  • RHS meaning: gradient of the log-probability of the action w.r.t. $\theta$, multiplied by return $R_t(\tau)$.
  • Eq. 1.5 states: gradient of objective = expected sum of (gradients of log action-probabilities × corresponding returns).
  • Full derivation given in Box 1.1.

Mechanism: policy gradient is how action probabilities produced by the policy get changed.

  • If $R_t(\tau) > 0$ → probability of $\pi_\theta(a_t \mid s_t)$ is increased.
  • If $R_t(\tau) < 0$ → probability of $\pi_\theta(a_t \mid s_t)$ is decreased.
  • Over many updates (Eq. 1.4), policy learns to produce actions resulting in high $R_t(\tau)$.
