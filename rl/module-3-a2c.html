A2C Algorithm

Synchronous Advantage Actor-Critic: algorithm, pseudocode, and implementation details.

Overview

A2C (Advantage Actor-Critic) is a synchronous, on-policy algorithm that combines the actor-critic architecture with advantage estimation. It is the synchronous version of A3C (Asynchronous Advantage Actor-Critic), where multiple parallel workers collect experience and update a shared model synchronously.

Key idea: A2C maintains both a policy network (actor) and a value network (critic). The actor is updated using the advantage estimate, and the critic is updated to minimize the TD error. Multiple parallel workers collect experience simultaneously, and gradients are accumulated synchronously before updating.

Architecture

  • Actor ($\pi_\theta$): maps states to action probabilities.
  • Critic ($V_\phi$): maps states to value estimates.
  • Advantage estimator: typically uses $n$-step returns or GAE.
  • Parallel workers: $N$ independent environments run in parallel, each collecting $T$ steps of experience before a synchronized update.

Algorithm Pseudocode

Algorithm 3: A2C (Synchronous Advantage Actor-Critic)
1: Initialize actor $\pi_\theta$ and critic $V_\phi$ with random weights 2: Initialize learning rates $\alpha_\pi$, $\alpha_V$ 3: for episode = 0, ..., MAX_EPISODE do 4: Reset all $N$ parallel environments 5: for step = 0, ..., $T-1$ do 6: for each worker $i$ in parallel do 7: Sample $a_t^{(i)} \sim \pi_\theta(\cdot\mid s_t^{(i)})$ 8: Execute $a_t^{(i)}$, observe $r_t^{(i)}$, $s_{t+1}^{(i)}$ 9: end for 10: end for 11: Compute advantages $\hat{A}_t^{(i)}$ for each worker $i$ and step $t$ 12: $\mathcal{L}_\pi = -\frac{1}{NT}\sum_{i=1}^{N}\sum_{t=0}^{T-1} \hat{A}_t^{(i)} \log \pi_\theta(a_t^{(i)}\mid s_t^{(i)})$ 13: $\mathcal{L}_V = \frac{1}{NT}\sum_{i=1}^{N}\sum_{t=0}^{T-1} \big(\hat{A}_t^{(i)}\big)^2$ 14: $\theta \leftarrow \theta - \alpha_\pi \nabla_\theta \mathcal{L}_\pi$ 15: $\phi \leftarrow \phi - \alpha_V \nabla_\phi \mathcal{L}_V$ 16: end for

Loss Functions

Actor Loss

The actor is trained to maximize the expected advantage. This is equivalent to minimizing the negative log-probability weighted by the advantage:

$$\mathcal{L}_\pi = -\frac{1}{NT}\sum_{i=1}^{N}\sum_{t=0}^{T-1} \hat{A}_t^{(i)} \log \pi_\theta(a_t^{(i)}\mid s_t^{(i)})$$

This is the same policy gradient objective as REINFORCE, but with the advantage replacing the Monte Carlo return.

Critic Loss

The critic is trained to minimize the squared TD error (or equivalently, the squared advantage, since $\hat{A}_t$ is the TD error):

$$\mathcal{L}_V = \frac{1}{NT}\sum_{i=1}^{N}\sum_{t=0}^{T-1} \big(\hat{A}_t^{(i)}\big)^2$$

This is the mean squared error between the target value $r_t + \gamma V_\phi(s_{t+1})$ and the current estimate $V_\phi(s_t)$.

Advantage Computation

A2C typically uses $n$-step returns for advantage estimation. For each worker $i$ at time step $t$:

$$\hat{A}_t^{(i)} = \sum_{k=0}^{n-1} \gamma^k r_{t+k}^{(i)} + \gamma^n V_\phi(s_{t+n}^{(i)}) - V_\phi(s_t^{(i)})$$

where $n$ is the number of steps collected before bootstrapping (often $n=5$ or $n=20$). This provides a good bias-variance trade-off.

A2C vs. A3C

PropertyA2CA3C
Update styleSynchronousAsynchronous
WorkersCollect experience, then update togetherEach worker updates independently
Gradient accumulationAccumulated, then applied onceEach worker applies gradients immediately
Hardware efficiencyBetter GPU utilizationMore CPU-efficient
DeterminismDeterministic (same seed → same result)Non-deterministic (race conditions)
Note: In practice, A2C often performs as well as or better than A3C, while being simpler to implement and debug. The synchronous update eliminates the race conditions inherent in A3C's asynchronous updates.

Entropy Bonus

To encourage exploration and prevent premature convergence to deterministic policies, A2C often adds an entropy bonus to the actor loss:

$$\mathcal{L}_\pi = -\frac{1}{NT}\sum_{i=1}^{N}\sum_{t=0}^{T-1} \hat{A}_t^{(i)} \log \pi_\theta(a_t^{(i)}\mid s_t^{(i)}) - \beta H(\pi_\theta(\cdot\mid s_t^{(i)}))$$

where $H(\pi_\theta(\cdot\mid s)) = -\sum_a \pi_\theta(a\mid s) \log \pi_\theta(a\mid s)$ is the policy entropy, and $\beta$ is a small coefficient (e.g., $\beta=0.01$). The entropy bonus penalizes overly confident policies, encouraging the agent to keep exploring.

Implementation Notes

  • Parameter sharing: The actor and critic can share lower-level layers (e.g., convolutional layers for image inputs) with separate output heads for the policy and value.
  • Normalization: Advantages are often normalized across the batch (subtract mean, divide by standard deviation) to stabilize training.
  • Learning rates: Typically $\alpha_\pi = \alpha_V = 3\times 10^{-4}$ for Adam optimizer.
  • Number of workers: Common choices are $N=8$ or $N=16$ parallel environments.
  • Rollout length: $T=5$ to $T=20$ steps before each update.
