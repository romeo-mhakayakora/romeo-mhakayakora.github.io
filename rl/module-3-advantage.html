Derivation of Generalized Advantage Estimation (GAE)

From n-step advantage estimators to the clean, efficient GAE form via telescoping cancellation of TD errors.

Contents

  • 03.1 A2C Introduction
  • 03.2 Derivation of GAE
  • 03.3 A2C Algorithm

The derivation shows how GAE arrives at its final, simple form. It starts from the raw n-step advantage estimators and ends with a clean weighted sum of TD errors.

N-step advantage estimator

Combining the n-step estimate for $Q^\pi$ with $V^\pi(s_t)$:

$$A^\pi_{\text{NSTEP}}(s_t,a_t) = Q^\pi(s_t,a_t) - V^\pi(s_t) \tag{3.8}$$

$$\approx r_t + \gamma r_{t+1} + \cdots + \gamma^n r_{t+n} + \gamma^{n+1}V^\pi(s_{t+n+1}) - V^\pi(s_t) \tag{3.9}$$

  • $n$ = number of steps of actual rewards used; controls the amount of variance in the advantage estimate; a hyperparameter that must be tuned.
  • Small $n$ → lower variance, higher bias. Large $n$ → higher variance, lower bias.

Step 1 — Define the n-step advantage estimator

$A_t^\pi(n)$ denotes the advantage estimate calculated using n-step forward returns:

$$A_t^\pi(1) = r_t + \gamma V^\pi(s_{t+1}) - V^\pi(s_t)$$

$$A_t^\pi(2) = r_t + \gamma r_{t+1} + \gamma^2 V^\pi(s_{t+2}) - V^\pi(s_t)$$

$$A_t^\pi(3) = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \gamma^3 V^\pi(s_{t+3}) - V^\pi(s_t) \tag{3.12}$$

Step 2 — The raw GAE definition is hard to work with

GAE is defined as an exponentially weighted average of all n-step forward return advantage estimators, $n = 1, 2, 3, \ldots$:

$$A^\pi_{\text{GAE}}(s_t,a_t) = (1-\lambda)\big(A_t^\pi(1) + \lambda A_t^\pi(2) + \lambda^2 A_t^\pi(3) + \cdots\big), \quad \lambda \in [0,1] \tag{3.13}$$

  • This is mathematically correct, but not practical: it involves an infinite sum of n-step terms, each of which is itself a growing expression involving many value-function terms.

Step 3 — Introduce the TD error $\delta_t$

Schulman et al. introduce a variable $\delta$ to simplify this:

$$\delta_t = r_t + \gamma V^\pi(s_{t+1}) - V^\pi(s_t) \tag{3.14}$$

  • Notice $r_t + \gamma V^\pi(s_{t+1})$ is exactly the 1-step estimator for $Q^\pi(s_t,a_t)$.
  • So $\delta_t$ is exactly the 1-step advantage estimator: $\delta_t = A_t^\pi(1)$.
  • $\delta_t$ represents the immediate advantage at time step $t$, computed using only a 1-step return.
  • Plain English: $\delta_t$ is just the 1-step advantage/TD error at that individual timestep — a fresh "how surprising was this one step" measurement computed independently at every $t$. The n-step advantage is then built by stacking these individual per-timestep measurements together with discounting.

Step 4 — Rewrite the n-step advantages using $\delta$

$$A_t^\pi(1) = \delta_t$$

$$A_t^\pi(2) = \delta_t + \gamma\delta_{t+1}$$

$$A_t^\pi(3) = \delta_t + \gamma\delta_{t+1} + \gamma^2\delta_{t+2} \tag{3.15–3.16}$$

Why this works — the telescoping cancellation (worked example for n=2)

Start with the 2-step estimator (Eq. 3.12):

$$A_t^\pi(2) = r_t + \gamma r_{t+1} + \gamma^2 V^\pi(s_{t+2}) - V^\pi(s_t)$$

Expand using the definitions of $\delta_t$ and $\delta_{t+1}$:

$$\delta_t = r_t + \gamma V^\pi(s_{t+1}) - V^\pi(s_t)$$

$$\delta_{t+1} = r_{t+1} + \gamma V^\pi(s_{t+2}) - V^\pi(s_{t+1})$$

Compute $\delta_t + \gamma\delta_{t+1}$:

Expansion:
$\delta_t + \gamma\delta_{t+1} = \big[r_t + \gamma V^\pi(s_{t+1}) - V^\pi(s_t)\big] + \gamma\big[r_{t+1} + \gamma V^\pi(s_{t+2}) - V^\pi(s_{t+1})\big]$

Distribute:

$$= r_t + \gamma V^\pi(s_{t+1}) - V^\pi(s_t) + \gamma r_{t+1} + \gamma^2 V^\pi(s_{t+2}) - \gamma V^\pi(s_{t+1})$$

Cancel: $+\gamma V^\pi(s_{t+1})$ and $-\gamma V^\pi(s_{t+1})$ cancel each other:

$$= r_t + \gamma r_{t+1} + \gamma^2 V^\pi(s_{t+2}) - V^\pi(s_t)$$

This is exactly $A_t^\pi(2)$ from Eq. 3.12. ✓

The same pattern for n=3

Expand $\delta_t + \gamma\delta_{t+1} + \gamma^2\delta_{t+2}$:

TermExpansion
$\delta_t$$r_t + \gamma V^\pi(s_{t+1}) - V^\pi(s_t)$
$\gamma\delta_{t+1}$$\gamma r_{t+1} + \gamma^2 V^\pi(s_{t+2}) - \gamma V^\pi(s_{t+1})$
$\gamma^2\delta_{t+2}$$\gamma^2 r_{t+2} + \gamma^3 V^\pi(s_{t+3}) - \gamma^2 V^\pi(s_{t+2})$

Cancellations:

  • $\gamma V^\pi(s_{t+1})$ cancels with $-\gamma V^\pi(s_{t+1})$
  • $\gamma^2 V^\pi(s_{t+2})$ cancels with $-\gamma^2 V^\pi(s_{t+2})$

What remains:

$$r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \gamma^3 V^\pi(s_{t+3}) - V^\pi(s_t) = A_t^\pi(3) \checkmark$$

The general pattern

For any $n$:

$$A_t^\pi(n) = \sum_{i=0}^{n-1} \gamma^i \delta_{t+i}$$

  • The intermediate value estimates $V^\pi(s_{t+1}), V^\pi(s_{t+2}), \ldots, V^\pi(s_{t+n-1})$ all telescope away (cancel in pairs).
  • What's left: the initial value estimate $-V^\pi(s_t)$, the final bootstrapped value $\gamma^n V^\pi(s_{t+n})$, and all the rewards in between.
  • Plain-English version: an n-step advantage is a sum of 1-step advantages (TD errors), each discounted by $\gamma$ according to how far into the future it is. Each $\delta_{t+i}$ "hands off" its value estimate to the next $\delta$, and that next $\delta$ "returns" it — so the intermediate $V^\pi$ terms wash out, leaving only the step-by-step "surprises," stacked and discounted.
  • This is why the $\delta$-formulation is powerful: it turns a messy expression with nested value functions into a simple sum of independent TD errors.

Step 5 — Substitute back into GAE and simplify

Writing the advantage function in the $\delta$-form (Eq. 3.15–3.16) shows it consists of multiple 1-step advantages, weighted exponentially by $\gamma$ as the time step increases.

Substituting $A_t^\pi(n) = \sum_{i=0}^{n-1}\gamma^i\delta_{t+i}$ into the raw GAE definition (Eq. 3.13) and simplifying yields the final, elegant form:

$$A^\pi_{\text{GAE}}(s_t,a_t) = \sum_{\ell=0}^{\infty} (\gamma\lambda)^\ell \delta_{t+\ell} \tag{3.17}$$

  • Both GAE and the n-step advantage estimator include the discount factor $\gamma$ (controls how much later rewards are discounted).
  • Both also have a parameter controlling the bias–variance trade-off: $n$ for the n-step advantage estimator, $\lambda$ for GAE.
  • What's gained with GAE: $n$ is a hard choice (it precisely determines the exact cutoff point between real rewards and the value-function estimate). $\lambda$ is a soft choice — smaller $\lambda$ weights the $V$-function baseline more heavily, larger $\lambda$ weights the actual rewards more heavily; the transition is smooth rather than a hard cutoff.
