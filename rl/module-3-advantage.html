The Advantage Function

The advantage function measures how much better an action $a$ is compared to the average action in state $s$:

$$A^\pi(s,a) = Q^\pi(s,a) - V^\pi(s)$$

If $A^\pi(s,a) > 0$, the action is better than average; if $A^\pi(s,a) < 0$, it is worse than average. Using the advantage as the coefficient in the policy gradient update naturally centers the returns, providing an action-dependent baseline.

Why Use Advantage?

Recall from REINFORCE that without a baseline, the policy gradient update is:

$$\nabla_\theta J(\pi_\theta) \approx \sum_{t=0}^{T} R_t(\tau) \nabla_\theta \log \pi_\theta(a_t\mid s_t)$$

The return $R_t(\tau)$ is always positive for typical environments (or always negative for costs). This means all actions in a trajectory get encouraged or discouraged uniformly, even though some actions may have been good and others bad.

With advantage: actions that are better than average get positive weight; actions that are worse than average get negative weight. This aligns the gradient update with the relative quality of each action, not just the absolute magnitude of the return.

Estimating Advantage with TD Error

The simplest advantage estimate is the TD error:

$$\hat{A}_t = \delta_t = r_t + \gamma V_\phi(s_{t+1}) - V_\phi(s_t)$$

This is a biased estimator of the advantage function because it relies on the critic's estimate $V_\phi$, which may be inaccurate. However, the bias decreases as the critic becomes more accurate, and the variance is much lower than using Monte Carlo returns.

Why TD-error is an advantage estimate

Recall that $Q^\pi(s_t,a_t) = r_t + \gamma V^\pi(s_{t+1})$. Substituting into the advantage definition:

$$A^\pi(s_t,a_t) = r_t + \gamma V^\pi(s_{t+1}) - V^\pi(s_t)$$

When we approximate $V^\pi$ with $V_\phi$, we get exactly $\delta_t$. So TD error is an unbiased estimate of the advantage if we had the true value function. With a learned critic, it is biased but low-variance.

$n$-step Returns

We can trade off bias and variance by using $n$-step returns instead of 1-step TD:

  • 1-step return ($n=1$): $\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$ — low variance, higher bias.
  • $n$-step return: $\hat{A}_t^{(n)} = \sum_{k=0}^{n-1} \gamma^k r_{t+k} + \gamma^n V(s_{t+n}) - V(s_t)$ — more variance, less bias.
  • Monte Carlo return ($n=\infty$): $\hat{A}_t^{(\infty)} = R_t(\tau) - V(s_t)$ — highest variance, unbiased.

Generalized Advantage Estimation (GAE)

GAE combines all $n$-step returns in an exponentially weighted average, controlled by a parameter $\lambda \in [0,1]$:

$$\hat{A}_t^{\text{GAE}(\gamma,\lambda)} = \sum_{k=0}^{\infty} (\gamma\lambda)^k \delta_{t+k}$$

where $\delta_{t+k} = r_{t+k} + \gamma V_\phi(s_{t+k+1}) - V_\phi(s_{t+k})$ are the 1-step TD errors.

$\lambda$BehaviorBiasVariance
$\lambda=0$1-step TD advantageHigherLower
$\lambda=0.5$Balanced mixModerateModerate
$\lambda=1$Monte Carlo advantageLowHigh
GAE insight: GAE smoothly interpolates between low-variance (but high-bias) 1-step advantage estimates and high-variance (but low-bias) Monte Carlo advantage estimates. The $\lambda$ parameter lets practitioners control this trade-off directly.
